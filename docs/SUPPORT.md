# Support

Need help with DeepFinder? Here's where to go.

## 🐛 Found a Bug?

Search [existing issues](https://github.com/nadav/deepfinder/issues) first to see if it's already reported. If not, open a new issue using the Bug Report template and include:

- macOS version
- DeepFinder version (`deepfinder --version`)
- Steps to reproduce
- What you expected vs. what happened
- Output of `deepfinder daemon status` (if applicable)

## 💡 Feature Request?

Open an issue with the Feature Request template. Describe:
- What you want to do
- Why current capabilities don't cover it
- How you'd expect it to work

## ❓ Questions?

- **Usage questions**: Check the [FAQ](how-to/faq.md) first — your question may already be answered
- **Troubleshooting**: See the [Troubleshooting Guide](how-to/troubleshooting.md) organized by symptom
- **Still stuck?** Open a [GitHub Discussion](https://github.com/nadav/deepfinder/discussions) or an issue with the Question template

## 🔒 Security Vulnerability?

Do NOT open a public issue. See [SECURITY.md](../SECURITY.md) for responsible disclosure instructions.

## 🤝 Want to Contribute?

See [CONTRIBUTING.md](../CONTRIBUTING.md) for setup instructions, code style, and the pull request process.

## 📊 Daemon Status

If you are experiencing issues with search results or daemon connectivity, run:

```bash
deepfinder daemon status
```

This prints the daemon PID, uptime, index state, file count, and memory usage. Include this output when reporting bugs -- it dramatically speeds up diagnosis.

## 📧 Direct Contact

For sensitive issues that can't go through GitHub (security, legal, private bug reports):
- Email: nadav@nadav.com.cn
- Response time: typically within 48 hours

---

**DeepFinder is a solo-maintainer open source project.** Responses may not be instant, but every issue is read and acknowledged.
