## Architecture

```
CLI -> Unix socket IPC -> Daemon -> SearchCoordinator -> InMemoryIndex (actor) -> results
```

- **Daemon + thin CLI (Everything model)**: daemon holds the full in-memory index
- **IPC**: 4-byte length prefix + JSON body over Unix domain socket at `~/.deep-finder/ipc.sock`
- **Persistence**: SQLite WAL at `~/.deep-finder/index.db`, batch writes every 5s or 100 changes
- **Index state machine**: stale -> verifying -> live

## Known issues

- PID file has a TOCTOU race condition between check and bind
- `try?` sites need auditing for silent failure swallowing
- SQLite database lacks encryption at rest
