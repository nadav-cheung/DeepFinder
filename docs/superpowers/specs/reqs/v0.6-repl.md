### REQ-0.6-01 REPL 交互循环 ✅ 🖥️ P0

**用户故事**

作为「用户」，我希望在终端输入 `deepfinder`（无参数）就能进入交互模式，持续发起多次搜索而无需每次重新连接 daemon，以便连续定位多个文件而不必反复启动进程。

**验收标准（Given / When / Then）**

- **AC1 — 进入交互模式**
  - **Given** 用户在终端
  - **When** 输入 `deepfinder`（无参数）
  - **Then** 启动 REPL 交互模式

- **AC2 — readline prompt**
  - **Given** REPL 已启动
  - **When** 等待输入
  - **Then** 使用 Darwin.readline（libedit）显示 prompt `> `（空格后缀）

- **AC3 — 查询 → 结果**
  - **Given** REPL prompt 显示中
  - **When** 用户输入查询
  - **Then** 经 IPC 查询并由 TerminalFormatter 输出结果

- **AC4 — 连续查询**
  - **Given** 上一次查询结果已输出
  - **When** 用户继续输入下一个查询
  - **Then** 新结果替换旧结果

- **AC5 — Ctrl+C 中断当前查询**
  - **Given** 查询进行中
  - **When** 用户按 Ctrl+C
  - **Then** 中断当前查询，但不退出 REPL

- **AC6 — Ctrl+D / :quit 退出**
  - **Given** REPL 运行中
  - **When** 用户按 Ctrl+D 或输入 `:quit`
  - **Then** 退出 REPL

- **AC7 — 历史持久化**
  - **Given** 用户发起过查询
  - **When** REPL 退出
  - **Then** 历史持久化到 `~/.deep-finder/history`（最近 1000 条）

- **AC8 — Tab 补全**
  - **Given** REPL 输入中
  - **When** 用户按 Tab
  - **Then** 提供命令名补全 + 文件路径补全（基于索引查询）

**实现**：`Sources/CLI/REPL.swift` · **测试**：`Tests/CLITests/REPLTests.swift`

### REQ-0.6-02 REPL 命令 ✅ 🖥️ P0

**用户故事**

作为「用户」，我希望在 REPL 中输入冒号前缀的元命令来查看索引统计、打开结果或在 Finder 中显示，以便获得搜索之外的辅助操作能力。

**验收标准（Given / When / Then）**

- **AC1 — :help**
  - **Given** REPL 运行中
  - **When** 输入 `:help`
  - **Then** 显示所有命令及用法

- **AC2 — :quit / :q**
  - **Given** REPL 运行中
  - **When** 输入 `:quit` 或 `:q`
  - **Then** 退出 REPL

- **AC3 — :stats**
  - **Given** REPL 运行中
  - **When** 输入 `:stats`
  - **Then** 显示索引统计（文件数、索引大小、daemon 状态、索引状态机状态）

- **AC4 — :config KEY [VALUE]**
  - **Given** REPL 运行中
  - **When** 输入 `:config KEY [VALUE]`
  - **Then** 通过 IPC 获取/设置配置项

- **AC5 — :open N**
  - **Given** 搜索结果列表存在
  - **When** 输入 `:open N`
  - **Then** 用默认应用打开第 N 个结果（NSWorkspace.shared.open）

- **AC6 — :reveal N**
  - **Given** 搜索结果列表存在
  - **When** 输入 `:reveal N`
  - **Then** 在 Finder 中显示第 N 个结果（NSWorkspace.shared.selectFile）

- **AC7 — :daemon**
  - **Given** REPL 运行中
  - **When** 输入 `:daemon`
  - **Then** 显示 daemon 状态信息（PID、运行时长、连接数）

- **AC8 — 命令不区分大小写**
  - **Given** 任意元命令
  - **When** 以不同大小写输入
  - **Then** 均被正确识别

- **AC9 — 无效命令提示**
  - **Given** REPL 运行中
  - **When** 输入未知的冒号命令
  - **Then** 友好提示 "未知命令，输入 :help 查看所有命令"

**实现**：`Sources/CLI/REPLCommands.swift` · **测试**：`Tests/CLITests/REPLTests.swift`

### REQ-0.6-03 REPL 历史与导航 ✅ 🖥️ P1

**用户故事**

作为「用户」，我希望通过上箭头浏览历史查询、用 Ctrl+R 搜索历史、用 Tab 补全命令或文件路径，以便快速复用之前的搜索而无需重新输入。

**验收标准（Given / When / Then）**

- **AC1 — 历史持久化**
  - **Given** 用户发起过查询
  - **When** 通过 readline 操作历史
  - **Then** 历史持久化写入 `~/.deep-finder/history`

- **AC2 — 上下箭头浏览历史**
  - **Given** 历史记录非空
  - **When** 用户按上/下箭头
  - **Then** 浏览上一个/下一个历史查询（readline 内置）

- **AC3 — Ctrl+R 搜索历史**
  - **Given** 历史记录非空
  - **When** 用户按 Ctrl+R
  - **Then** 进行 readline 内置的 incremental search

- **AC4 — 命令名补全**
  - **Given** 用户输入命令名前缀（如 `:st`）
  - **When** 按 Tab
  - **Then** 补全为 `:stats`

- **AC5 — 文件路径补全**
  - **Given** 用户输入部分路径
  - **When** 按 Tab
  - **Then** 基于索引查询补全文件路径

- **AC6 — 历史去重**
  - **Given** 用户连续输入相同查询
  - **When** 写入历史
  - **Then** 连续重复查询不记录

**实现**：`Sources/CLI/REPLHistory.swift` · **测试**：`Tests/CLITests/REPLHistoryTests.swift`
