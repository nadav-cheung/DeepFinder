## 设计哲学

**"Speed That Feels Invisible"** — 最快 UI 是用户感觉不到的 UI。每次交互 < 100ms，键盘优先，零认知负担。视觉打磨（glow、动画）增强速度感而非削弱。

参考应用（按影响权重排序）：
1. Windows Everything — 亚秒级原始搜索，海量结果处理
2. Raycast — 键盘优先，action panel，简约但精致
3. macOS Spotlight — Liquid Glass，原生 macOS 集成
4. Alfred — 模糊搜索，丰富功能深度

## 架构决策

### REQ-3.2-06 合并入 REQ-3.2-37

原 REQ-3.2-06（空状态键盘提示标签）被 REQ-3.2-37（bottom action bar）吸收。Action bar 始终可见（不仅空状态），遵循 Raycast 模式，提供更好的可发现性。空状态改为显示搜索历史或零结果建议。

### IntelligenceGlow 推迟到 Phase 3

4 层 glow 升级视觉效果出色但功能上是锦上添花。以"速度 & 效率"为优先，键盘导航和结果列表性能先发布。当前单层 glow 可接受。

### 信任 LazyVStack — 无手动虚拟化

基于最佳实践调研：macOS 26 LazyVStack 有双向惰性加载及内置回收。不应手动虚拟化（"buffer ±5 rows"）。改用 `.equatable()` + 后台 AttributedString + 固定行高。

### Liquid Glass — 仅导航层

Apple WWDC25 指导：`.glassEffect()` 仅用于导航 chrome（sidebar、toolbar、panel），不应用于结果行（内容层）。搜索面板背景用 glass，行用标准列表样式。

### 键盘栈优先级

```
↑/↓              → selection (primary)
Ctrl+N/Ctrl+P    → selection (alias, Emacs compat)
Type-to-select   → press letter jumps to first matching filename, same-letter cycles
⌥↑/⌥↓           → page jump (~20 lines)
⌘↑/⌘↓           → group jump (when categories active)
Home/End         → first/last result
```
