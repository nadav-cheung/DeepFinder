## 设计哲学

**"Speed That Feels Invisible"** — 最快 UI 是用户感觉不到的 UI。每次交互 < 100ms，键盘优先，零认知负担。视觉打磨（glow、动画）增强速度感而非削弱。

参考应用（按影响权重排序）：
1. Windows Everything — 亚秒级原始搜索，海量结果处理
2. Raycast — 键盘优先，action panel，简约但精致
3. macOS Spotlight — Liquid Glass，原生 macOS 集成
4. Alfred — 模糊搜索，丰富功能深度

### REQ-3.2-04 Spinner 仅 AI 查询时显示 ✅ ⚡ P0

**用户故事**

作为「用户」，我希望本地搜索（<50ms）不出现加载动画，以便避免不必要的视觉干扰；仅 AI 查询耗时较长时才用 spinner 反馈等待状态。

**验收标准（Given / When / Then）**

- **AC1 — 本地查询无 spinner**
  - **Given** 用户发起本地搜索且耗时 < 50ms
  - **When** 查询执行
  - **Then** 界面不显示 spinner

- **AC2 — AI 查询显示 spinner**
  - **Given** 用户发起 AI 查询（耗时较长）
  - **When** 查询执行
  - **Then** spinner 正常显示以反馈等待状态

- **AC3 — 类型切换即时响应**
  - **Given** 用户在本地查询与 AI 查询之间切换
  - **When** 查询类型变化
  - **Then** spinner 状态即时响应（显示或隐藏）

**实现**：`Sources/GUI/SearchViewModel.swift`, `Sources/GUI/SearchPanelView.swift` · **测试**：`Tests/GUITests/SearchPanelTests.swift`

### REQ-3.2-07 自适应面板高度 ✅ ⚡ P0

**用户故事**

作为「用户」，我希望在不同屏幕尺寸（13"/15"/外接显示器）上使用 DeepFinder 时面板自动适应屏幕高度，以便展示尽可能多的结果。

**验收标准（Given / When / Then）**

- **AC1 — 高度公式**
  - **Given** 任意屏幕尺寸
  - **When** 计算面板高度
  - **Then** 面板高度 = `min(searchBarHeight + minVisibleRows × rowHeight, screenH × (1 - topPaddingRatio) - bottomMargin)`（理想高度 = 搜索栏 + 20 行 × 40pt；屏幕不足时按可用高度夹取）

- **AC2 — 15" MacBook 可见行数**
  - **Given** 15" MacBook 屏幕
  - **When** 面板展开
  - **Then** ≥ 15 行结果可见（`minVisibleRows = 20` 保证此目标）

- **AC3 — 外接显示器上限**
  - **Given** 外接大屏显示器
  - **When** 面板展开
  - **Then** 面板高度不超过理想高度（搜索栏 + 20 行），不撑满整屏

**实现**：`Sources/GUI/SearchPanelView.swift` · **测试**：`Tests/GUITests/SearchPanelTests.swift`

- **AC1 — ↑↓ 选择**
  - **Given** 结果列表已渲染且有选中项

- **AC2 — Ctrl+N/P Emacs 别名**

- **AC4 — 首尾循环**
  - **Given** 选中末尾或首行结果

  - **Given** 结果列表
  - **When** 仅使用键盘（↑/↓、Ctrl+N/P、Enter/⌘Enter、Space、⌘C、⌘I、⌘K、Tab）

  - **Given** 1000 行结果目标

- **AC3 — 主线程不卡顿**
  - **Given** 大量结果滚动
  - **When** 触发高亮计算
  - **Then** 主线程不因高亮计算而卡顿

## 架构决策

#### REQ-3.2-06 合并入 REQ-3.2-37

原 REQ-3.2-06（空状态键盘提示标签）被 REQ-3.2-37（bottom action bar）吸收。Action bar 始终可见（不仅空状态），遵循 Raycast 模式，提供更好的可发现性。空状态改为显示搜索历史或零结果建议。

### IntelligenceGlow 推迟到 Phase 3

4 层 glow 升级视觉效果出色但功能上是锦上添花。以"速度 & 效率"为优先，键盘导航和结果列表性能先发布。当前单层 glow 可接受。

### 信任 LazyVStack — 无手动虚拟化

基于最佳实践调研：macOS 26 LazyVStack 有双向惰性加载及内置回收。不应手动虚拟化（"buffer ±5 rows"）。改用 `.equatable()` + 后台 AttributedString + 固定行高。

### Liquid Glass — 仅导航层

Apple WWDC25 指导：`.glassEffect()` 仅用于导航 chrome（sidebar、toolbar、panel），不应用于结果行（内容层）。搜索面板背景用 glass，行用标准列表样式。

### 键盘栈优先级

```
↑/↓              → selection (primary)
Ctrl+N/Ctrl+P    → selection (alias, Emacs compat)
Type-to-select   → press letter jumps to first matching filename, same-letter cycles
⌥↑/⌥↓           → page jump (~20 lines)
⌘↑/⌘↓           → group jump (when categories active)
Home/End         → first/last result
```
