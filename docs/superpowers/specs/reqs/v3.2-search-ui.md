## 设计哲学

**"Speed That Feels Invisible"** — 最快 UI 是用户感觉不到的 UI。每次交互 < 100ms，键盘优先，零认知负担。视觉打磨（glow、动画）增强速度感而非削弱。

参考应用（按影响权重排序）：
1. Windows Everything — 亚秒级原始搜索，海量结果处理
2. Raycast — 键盘优先，action panel，简约但精致
3. macOS Spotlight — Liquid Glass，原生 macOS 集成
4. Alfred — 模糊搜索，丰富功能深度

## Phase 1 — Core Speed (15 REQs, P0)

核心用户流程：热键唤起 → 面板打开 → 输入 → 结果出现 → 导航 → 操作 → 面板关闭。

### REQ-3.2-04 Spinner 仅 AI 查询时显示，本地查询（<50ms）隐藏 ✅ ⚡ P0

**用户场景**: 用户执行本地搜索时不需要加载动画，避免不必要的视觉干扰。仅 AI 查询耗时较长时才需要 spinner 反馈等待状态。

**验收标准**:
- [ ] 本地搜索（<50ms）无 spinner 显示
- [ ] AI 查询时 spinner 正常显示
- [ ] 查询类型切换时 spinner 状态即时响应

### REQ-3.2-07 自适应面板高度 ✅ ⚡ P0

**用户场景**: 用户在不同屏幕尺寸（13"/15"/外接显示器）上使用 DeepFinder 时，面板应自动适应屏幕高度以展示尽可能多的结果。

**验收标准**:
- [ ] 面板高度 = `min(screenH - searchBox - margins, 800pt)`
- [ ] 15" MacBook 上 ≥ 15 行结果可见
- [ ] 外接显示器上最大化利用但不超出 800pt 上限

### REQ-3.2-08 ↑↓ 选择与 Ctrl+N/P 别名（Emacs 兼容）及 type-to-select ✅ ⚡ P0

**用户场景**: Emacs 用户习惯 Ctrl+N/P 进行上下移动；所有用户希望通过键盘完全操控结果列表，包括输入字母快速跳转到匹配文件名。

**验收标准**:
- [ ] ↑↓ 键选择上/下一个结果
- [ ] Ctrl+N/Ctrl+P 等效于 ↓/↑（Emacs 兼容）
- [ ] 输入字母跳转到第一个匹配文件名，同字母循环
- [ ] 首尾循环：末尾按 ↓ 回首位，首位按 ↑ 到尾行
- [ ] 纯键盘可完成所有结果导航

### REQ-3.2-09 ScrollViewReader 自动滚动与平滑动画 ✅ ⚡ P0

**用户场景**: 用户通过键盘导航结果时，选中行应始终在可视区域内，且滚动动画自然流畅不突兀。

**验收标准**:
- [ ] ScrollViewReader 自动将选中行滚动到可视区域
- [ ] 滚动动画使用 `.easeOut(0.15s)`
- [ ] 选中行靠上 1/3 位置，提供上下文预览
- [ ] 快速导航时无视觉跳动

### REQ-3.2-10 ⌥↑/⌥↓ 翻页导航 ✅ ⚡ P0

**用户场景**: 用户面对数百条搜索结果时，需要按页快速跳转，而非逐行滚动到底。

**验收标准**:
- [ ] ⌥↑ 向上翻页约 20 行
- [ ] ⌥↓ 向下翻页约 20 行
- [ ] 翻页边界处理正确（不满一页时跳至首/尾）

### REQ-3.2-12 AttributedString 后台线程高亮计算 ✅ ⚡ P0

**用户场景**: 搜索结果中匹配关键词的高亮渲染不应阻塞主线程，用户滚动 1000+ 结果时保持界面响应。

**验收标准**:
- [ ] AttributedString 高亮在后台线程计算
- [ ] 单次高亮计算 < 16ms（1000 行目标）
- [ ] 主线程不因高亮计算而卡顿

### REQ-3.2-14 LazyVStack + `.equatable()` 高性能虚拟化 ✅ ⚡ P0

**用户场景**: 用户在面对 10k+ 结果时流畅滚动浏览，帧率不受结果数量影响。

**验收标准**:
- [ ] LazyVStack 替代手动虚拟化管理
- [ ] ResultRowView 实现 `.equatable()`
- [ ] 固定行高 40pt
- [ ] 信任框架内置的回收机制，不自己实现 buffer ±5 rows
- [ ] 10k 结果时滚动 FPS ≥ 55

### REQ-3.2-17 选中动画快速按键跳过 ✅ ⚡ P0

**用户场景**: 用户快速连续按键导航时，中间帧的选中动画应被跳过，确保导航速度跟得上按键节奏，无视觉滞后感。

**验收标准**:
- [ ] 选中动画使用 `.easeInOut(0.12s)`
- [ ] 快速连续按键时跳过中间动画（0.1s 合并阈值）
- [ ] 快速导航无视觉延迟

**操作流程**: 用户按住 ↓ 键连续移动 → 动画被合并 → 选中指示器直接跳到最终位置。

### REQ-3.2-19 面板打开/关闭动画 ✅ ⚡ P0

**用户场景**: 用户按下全局热键时搜索面板快速弹出有弹性感，按 Esc 关闭时干脆利落不拖沓。

**验收标准**:
- [ ] 打开动画：spring 0.25s（cubic-bezier 0.2, 0.9, 0.3, 1.0）
- [ ] 关闭动画：easeOut 0.2s
- [ ] 打开感觉有弹性，关闭干脆

### REQ-3.2-24 Enter 打开文件并自动关闭面板 ✅ ⚡ P0

**用户场景**: 用户选中结果后按 Enter 直接打开文件，面板自动关闭完成搜索-操作闭环，无需额外步骤。

**验收标准**:
- [ ] Enter 键触发 `NSWorkspace.open(url)`
- [ ] 文件打开后面板自动关闭
- [ ] 高效完成搜索动作

### REQ-3.2-25 ⌘Enter 在 Finder 中定位文件 ✅ ⚡ P0

**用户场景**: 用户找到目标文件后需要查看其在文件夹中的位置，面板保持打开以继续搜索其他文件。

**验收标准**:
- [ ] ⌘Enter 触发 `NSWorkspace.activateFileViewerSelecting([url])`
- [ ] 面板保持打开不关闭
- [ ] 不打断用户搜索上下文

### REQ-3.2-26 Space/⌘Y Quick Look 预览 ✅ ⚡ P0

**用户场景**: 用户想快速预览文件内容而无需打开应用，空格键触发 Quick Look 并与结果列表选中项保持同步。

**验收标准**:
- [ ] Space 或 ⌘Y 触发 QLPreviewPanel
- [ ] 预览内容与结果列表选中项同步切换
- [ ] 再次按下关闭预览

### REQ-3.2-27 ⌘C 复制路径与内联 Toast ✅ ⚡ P0

**用户场景**: 开发者用户复制文件路径后需要即时反馈确认，避免粘贴到终端时才发现复制失败。

**验收标准**:
- [ ] ⌘C 复制当前选中文件的完整路径到剪贴板
- [ ] 面板内显示内联 toast 确认（如"已复制"）
- [ ] Toast 自动消失（约 1.5s）

### REQ-3.2-30 上下文感知 Esc ✅ ⚡ P0

**用户场景**: 用户在不同上下文（预览、详情面板、输入框有文字、面板）中按 Esc 时，应逐层退出而非一次性关闭所有层级。

**验收标准**:
- [ ] Quick Look 打开时 → Esc 关闭 QL
- [ ] 详情面板打开时（无 QL）→ Esc 关闭详情面板
- [ ] 搜索框有文字时 → Esc 清空文字
- [ ] 搜索框为空时 → Esc 关闭面板
- [ ] 分层退出符合用户预期

**操作流程**: QL 打开 → Esc 关闭 QL → 详情面板开 → Esc 关闭详情 → 文字存在 → Esc 清空 → 面板空 → Esc 关闭面板。

### REQ-3.2-36 CJK IME 稳定性 ✅ ⚡ P0

**用户场景**: 使用中文（拼音/五笔）、日文（假名）、韩文（韩字）输入法的用户，在输入过程中不应出现搜索闪烁、候选窗口消失或滚动位置跳动。

**验收标准**:
- [ ] IME 组合输入（marked text）期间不触发搜索
- [ ] 仅提交（committed text）后才触发搜索
- [ ] 候选窗口正常显示无闪烁
- [ ] 结果列表滚动位置在 IME 输入期间不变
- [ ] 中文/日文/韩文输入法完美兼容

**Phase 1 成功标准**: 用户可纯键盘完成：打开面板 → 输入 → 导航结果 → 打开/显示/预览/复制路径 → 关闭，全程 ≤ 2 秒。

## Phase 2 — Richness (9 REQs, P1)

搜索历史、分类、action panel、入场动画。在 Phase 1 速度基础上增加深度。

### REQ-3.2-01 动态 Placeholder 轮换 ✅ 🔧 P1

**用户场景**: 用户每次打开搜索面板时看到不同的搜索提示（如"搜索文件名…"、"试试 '*.pdf'…"、"输入关键词…"），帮助发现搜索功能和语法。

**验收标准**:
- [ ] Placeholder 从候选列表轮换
- [ ] 每次面板打开时切换到下一个提示
- [ ] 候选列表可配置（≥ 5 条）
- [ ] 不同提示文案实际出现

### REQ-3.2-02 搜索历史下拉（Raycast 模式） ✅ 🔧 P1

**用户场景**: 用户希望快速回溯之前的搜索，空输入框时按 ↑ 进入历史模式，符合 Raycast 用户的操作习惯。

**验收标准**:
- [ ] 空输入框 + ↑ 进入搜索历史下拉
- [ ] 历史列表显示最近 100 条搜索
- [ ] 选择历史项后执行搜索
- [ ] 搜索历史 JSON 持久化到 `~/.deep-finder/`
- [ ] 历史可回溯不丢失

### REQ-3.2-05 清除按钮透明度动画与 Esc 双重行为 ✅ 🔧 P1

**用户场景**: 用户清除搜索文字时希望看到平滑的视觉过渡，且 Esc 键在"清空文字"和"关闭面板"间行为区分明确。

**验收标准**:
- [ ] 清除按钮（×）显示/隐藏使用透明度渐变动画
- [ ] 有文字时 Esc 先清空文字
- [ ] 无文字时 Esc 关闭面板
- [ ] 清除操作有可见过渡

### REQ-3.2-13 结果计数 Footer ✅ 🔧 P1

**用户场景**: 用户希望了解搜索结果的总体规模以及当前显示进度，感知结果数量级。

**验收标准**:
- [ ] 结果列表底部显示总数（千位分隔符）
- [ ] 分页时显示"显示 M/N"格式
- [ ] 计数随搜索实时更新
- [ ] 用户感知结果规模

### REQ-3.2-18 结果出现渐入动画 ✅ 🔧 P1

**用户场景**: 搜索结果首次出现时应有流畅的渐入效果，让用户感知加载过渡而非内容突然闪现。

**验收标准**:
- [ ] 结果行使用 fade-in + slide-down 动画
- [ ] 前 5 行动画 stagger（错开延迟）
- [ ] 首次加载感觉流畅

### REQ-3.2-31 ⌘K Action Panel ✅ 🔧 P1

**用户场景**: 用户不记得某个操作的快捷键或菜单位置，按 ⌘K 打开 action panel，通过模糊搜索快速找到并执行操作。

**验收标准**:
- [ ] ⌘K 打开 action panel
- [ ] 操作列表支持模糊搜索过滤
- [ ] 键盘焦点捕获到 action panel 输入框
- [ ] 不可用的操作隐藏
- [ ] 每行显示对应快捷键
- [ ] 操作可发现无需菜单记忆

**操作流程**: ⌘K → 输入"路径" → 过滤出"复制路径 ⌘C" → Enter 执行。

### REQ-3.2-32 UTI 分组与 Sticky Headers ✅ 🔧 P1

**用户场景**: 用户面对大量混合类型结果时，希望按文件类型（文档、图片、音频等）分组查看，快速定位目标类别。

**验收标准**:
- [ ] 结果按 UTI 类型分组（文档/图片/音频/视频/代码/其他）
- [ ] 分组标题 sticky 吸顶
- [ ] 分组可折叠/展开
- [ ] "Other" 类别 > 5 项时自动折叠
- [ ] 大量结果有条理

### REQ-3.2-34 零结果友好提示 ✅ 🔧 P1

**用户场景**: 用户搜索无结果时不应面对空白页面，需要引导下一步操作（调整搜索词、尝试 AI 语义搜索等）。

**验收标准**:
- [ ] 零结果时显示友好提示文案
- [ ] 搜索建议以可点击按钮形式呈现
- [ ] AI 语义搜索作为主操作入口
- [ ] 搜索历史显示在下方
- [ ] 搜索失败时有下一步引导

### REQ-3.2-37 BottomActionBar 固定快捷键提示 ✅ 🔧 P1

**用户场景**: 新用户或不常用用户需要始终可见的操作快捷键提示，降低学习成本。替代原 REQ-3.2-06 键盘提示标签方案。

**验收标准**:
- [ ] 面板底部固定显示操作快捷键：`↵ 打开  ⌘↵ Finder  Space 预览  ⌘C 路径`
- [ ] 始终可见，不依赖空状态
- [ ] 吸收 REQ-3.2-06 的功能
- [ ] 遵循 Raycast 模式提供更好的可发现性

**Phase 2 成功标准**: 搜索历史加速重复查询。⌘K action panel 提供操作可发现性。结果分类组织大量结果。

## 架构决策

#### REQ-3.2-06 合并入 REQ-3.2-37

原 REQ-3.2-06（空状态键盘提示标签）被 REQ-3.2-37（bottom action bar）吸收。Action bar 始终可见（不仅空状态），遵循 Raycast 模式，提供更好的可发现性。空状态改为显示搜索历史或零结果建议。

### IntelligenceGlow 推迟到 Phase 3

4 层 glow 升级视觉效果出色但功能上是锦上添花。以"速度 & 效率"为优先，键盘导航和结果列表性能先发布。当前单层 glow 可接受。

### 信任 LazyVStack — 无手动虚拟化

基于最佳实践调研：macOS 26 LazyVStack 有双向惰性加载及内置回收。不应手动虚拟化（"buffer ±5 rows"）。改用 `.equatable()` + 后台 AttributedString + 固定行高。

### Liquid Glass — 仅导航层

Apple WWDC25 指导：`.glassEffect()` 仅用于导航 chrome（sidebar、toolbar、panel），不应用于结果行（内容层）。搜索面板背景用 glass，行用标准列表样式。

### 键盘栈优先级

```
↑/↓              → selection (primary)
Ctrl+N/Ctrl+P    → selection (alias, Emacs compat)
Type-to-select   → press letter jumps to first matching filename, same-letter cycles
⌥↑/⌥↓           → page jump (~20 lines)
⌘↑/⌘↓           → group jump (when categories active)
Home/End         → first/last result
```
