# v2.1 — 媒体元数据

7 项 REQ (P0: 4, P1: 3)

> **v2.1 在 v2.0 基础上增加媒体文件元数据提取和过滤**。索引构建时后台提取图片尺寸、音频标签、视频信息、PDF 元数据，存入索引。用户通过搜索过滤器查询：`width:>2560`、`artist:周杰伦`、`duration:>300`、`pdf-author:xxx`。所有提取完全本地，零外部依赖。

---

### REQ-2.1-01 MetadataExtractor 协议 📋 🖥️ P0

**用户场景**：用户无直接感知。协议设计确保新增媒体类型（如未来 RAW 图片）只需新增一个 Extractor 文件。

**验收标准**：
- [ ] `MetadataExtractor` 协议：`supportedExtensions: Set<String>`, `extract(url: URL) async -> ExtractedMetadata?`
- [ ] `ExtractedMetadata` 结构体：`[String: MetadataValue]` 字典，`MetadataValue` 为字符串/整数/浮点数/日期的枚举
- [ ] `MetadataExtractorRegistry` 管理所有 extractor 实例，按扩展名分发
- [ ] Swift 6 严格并发安全（Sendable）

---

### REQ-2.1-02 ImageMetadataExtractor（图片元数据）📋 🖥️ P0

**用户场景**：用户搜索 `width:>2560` 或 `height:>1440` → 找到所有高分辨率图片。无需记住文件名。

**验收标准**：
- [ ] ImageIO/CGImageSource 提取 EXIF 元数据
- [ ] 提取字段：width（像素宽）、height（像素高）、dpi、colorSpace、orientation、dateTaken
- [ ] 支持 JPG/PNG/HEIC/GIF/TIFF/BMP/WebP
- [ ] 文件损坏或无 EXIF 时返回 nil（不崩溃）
- [ ] 大文件（>50MB）异步提取，不阻塞索引

---

### REQ-2.1-03 AudioMetadataExtractor（音频元数据）📋 🖥️ P0

**用户场景**：用户搜索 `artist:周杰伦` 或 `duration:>300` → 找到特定歌手或特定长度的音频文件。

**验收标准**：
- [ ] AVFoundation 提取音频元数据
- [ ] 提取字段：title、artist、album、genre、duration（秒）、bitRate、sampleRate、channels、year
- [ ] 支持 MP3/AAC/M4A/WAV/FLAC/OGG/AIFF
- [ ] 文件无标签时仅返回 duration（AVFoundation 始终可获取时长）
- [ ] 损坏文件返回 nil

---

### REQ-2.1-04 VideoMetadataExtractor（视频元数据）📋 🖥️ P0

**用户场景**：用户搜索 `duration:>3600` 或 `width:3840` → 找到长视频或 4K 视频。

**验收标准**：
- [ ] AVFoundation 提取视频元数据
- [ ] 提取字段：width、height、duration（秒）、fps、codec、bitRate、audioCodec
- [ ] 支持 MP4/MOV/MKV/AVI/WMV/M4V
- [ ] 文件损坏时返回 nil
- [ ] 无视频轨的文件（纯音频）返回 nil

---

### REQ-2.1-05 PDFMetadataExtractor（PDF 元数据）📋 🖥️ P1

**用户场景**：用户搜索 `pdf-author:张三` 或 `pages:>50` → 找到特定作者的 PDF 或长文档。

**验收标准**：
- [ ] PDFKit 提取 PDF 元数据
- [ ] 提取字段：title、author、subject、creator、pageCount、creationDate、modificationDate、isEncrypted
- [ ] 加密 PDF 返回可提取字段（title/pageCount 可能可获取）
- [ ] 损坏 PDF 返回 nil

---

### REQ-2.1-06 MediaMetadataIndex（元数据索引）📋 🖥️ P1

**用户场景**：索引构建时后台提取元数据 → 存入内存索引 + SQLite 持久化 → 搜索时毫秒级过滤。

**验收标准**：
- [ ] `MediaMetadataIndex` actor 管理所有文件的元数据
- [ ] 元数据存入 FileRecord（扩展 FileRecord 增加 metadata 字典字段）
- [ ] SQLite 持久化：JSON 编码存储，批量写入（复用现有 IndexPersistence 批量逻辑）
- [ ] 索引构建时后台异步提取（低优先级 TaskGroup）
- [ ] 提取进度通过 IPC 上报（`:stats` 可查看进度）
- [ ] 已提取文件不重复提取（基于 mtime 检查）

---

### REQ-2.1-07 元数据搜索过滤器 📋 🖥️ P1

**用户场景**：用户使用 `width:>2560`、`artist:周杰伦`、`duration:>300`、`pdf-author:xxx` 等过滤语法搜索媒体文件。

**验收标准**：
- [ ] 扩展 SearchFilter 支持元数据字段过滤器
- [ ] 支持字段：width、height、duration、pages、artist、album、title、genre、pdf-author、pdf-title、fps、bitRate、codec
- [ ] 支持比较运算符：`:`（等于）、`:>`（大于）、`:<`（小于）、`:>=`、`:<=`（数值字段）
- [ ] 字符串字段支持子串匹配（如 `artist:周` 匹配 `周杰伦`）
- [ ] 无元数据的文件自动排除
- [ ] CLI 帮助文档（`:help filters`）列出所有可用过滤器
