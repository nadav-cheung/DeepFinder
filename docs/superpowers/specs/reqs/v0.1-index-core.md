### REQ-0.1-01 FileRecord 数据模型 ✅ 🖥️ P0

**用户故事**

作为「搜索引擎」，我希望有一个不可变、并发安全、NFC 规范化的文件记录类型，以便索引、持久化与搜索各层能以一致的语义引用同一个文件实体。

**验收标准（Given / When / Then）**

- **AC1 — 字段完整**
  - **Given** 一条已索引的文件记录
  - **When** 读取其字段
  - **Then** 包含 `id: UInt32`、`name`（NFC 规范化）、`originalName`、`path`、`parentPath`、`isDirectory`、`size`、`createdAt`、`modifiedAt`、`extension: String?`

- **AC2 — 并发安全**
  - **Given** Swift 6 严格并发模式
  - **When** 跨 actor / 跨任务传递该记录
  - **Then** 类型声明为 `Codable & Sendable`，编译通过且无并发告警

- **AC3 — 往返无损**
  - **Given** 任意 FileRecord
  - **When** 执行 JSON 编码后再解码
  - **Then** 所有字段相等（零丢失）

- **AC4 — 测试覆盖**
  - **Given** `FileRecordTests`
  - **When** 运行测试套件
  - **Then** 5 个用例全部通过

**实现**：`Sources/Index/FileRecord.swift` · **测试**：`Tests/IndexTests/FileRecordTests.swift`

> 备注：前向兼容字段 `isHidden`/`isSymlink`/`contentType` 已延期——v1.0–v3.0 不需要，待后续版本按需引入。

### REQ-0.1-02 Trie 前缀索引 ✅ 🖥️ P0

**用户故事**

作为「用户」，我希望每输入一个字符都能即时看到前缀匹配的文件，以便像 Everything 一样零延迟地逐步缩小定位范围。

**验收标准（Given / When / Then）**

- **AC1 — Unicode 粒度**
  - **Given** 中文 / 日文 / emoji 文件名已索引
  - **When** 以 Unicode scalar 为单位输入其前缀
  - **Then** 正确返回该文件

- **AC2 — 插入性能**
  - **Given** 100 万条记录
  - **When** 批量插入 Trie
  - **Then** 耗时 < 3 s

- **AC3 — 查询性能**
  - **Given** 100 万记录的 Trie
  - **When** 发起前缀查询
  - **Then** p99 延迟 < 2 ms

- **AC4 — 中文前缀**
  - **Given** `季度报告.pdf` 已索引
  - **When** 输入前缀「季」
  - **Then** 返回该文件

- **AC5 — 边界**
  - **Given** 空字符串或超长前缀
  - **When** 查询
  - **Then** 空串返回空结果；超长前缀返回空或精确匹配

**实现**：`Sources/Index/Trie.swift` · **测试**：`Tests/IndexTests/TrieTests.swift`

### REQ-0.1-03 FullSubstringMap ✅ 🖥️ P0

**用户故事**

作为「用户」，我希望输入任意子串（不限前缀）就能瞬间看到所有包含它的文件，以便不依赖文件名开头也能定位。

**验收标准（Given / When / Then）**

- **AC1 — 子串预建映射**
  - **Given** 文件名长度 ≤ 64 字符
  - **When** 索引该文件
  - **Then** 所有可能子串预建为「子串 → ID」映射，查询为 O(1)

- **AC2 — 查询性能**
  - **Given** 100 万记录、1 万次随机子串查询
  - **When** 查询
  - **Then** p99 延迟 < 1 ms

- **AC3 — 大小写不敏感**
  - **Given** 含大小写混合的文件名
  - **When** 查询
  - **Then** 存储与查询均小写化；结果保留原始大小写（`originalName`）

- **AC4 — 重复子串去重**
  - **Given** 文件名 `baaab` 含多处「aaa」
  - **When** 查询「aaa」
  - **Then** 该文件只返回一次

- **AC5 — 空串边界**
  - **Given** 空查询
  - **When** 查询
  - **Then** 返回空结果

**实现**：`Sources/Index/FullSubstringMap.swift` · **测试**：`Tests/IndexTests/FullSubstringMapTests.swift`

### REQ-0.1-05 PinyinIndex ✅ 🖥️ P0

**用户故事**

作为「中文用户」，我希望用拼音全拼或首字母缩写找到中文文件名，以便用英文键盘快速定位中文文件。

**验收标准（Given / When / Then）**

- **AC1 — 全拼匹配**
  - **Given** `季度报告.pdf` 已索引
  - **When** 输入「baogao」
  - **Then** 返回所有含「报告」的文件

- **AC2 — 首字母缩写**
  - **Given** 同上
  - **When** 输入「bg」
  - **Then** 返回同样的结果

- **AC3 — 混合输入**
  - **Given** `Q3报告.pdf` 已索引
  - **When** 输入「jdbg」
  - **Then** 经首字母逐字匹配命中 `季度报告.pdf`

- **AC4 — 中英混合**
  - **Given** 中英文混合文件名
  - **When** 提取拼音 token
  - **Then** 正确处理为「Q3baogao」式拼接；纯英文名不产生拼音 token（避免误匹配）

- **AC5 — locale 与声调**
  - **Given** 默认 locale
  - **When** 提取拼音
  - **Then** 使用 `CFLocaleWithIdentifier("zh-Hans")`；不存储声调标记；繁体经 `kCFStringTransformMandarinLatin` 转换

- **AC6 — 双 Trie 结构**
  - **Given** 索引构建
  - **When** 写入拼音索引
  - **Then** 同时构建全拼 Trie 与首字母 Trie

- **AC7 — 查询性能**
  - **Given** 100 万记录
  - **When** 拼音查询
  - **Then** p99 < 15 ms

**实现**：`Sources/Index/PinyinIndex.swift` · **测试**：`Tests/IndexTests/PinyinIndexTests.swift`

### REQ-0.1-06 InMemoryIndex（actor）✅ 🖥️ P0

**用户故事**

作为「系统」，我希望有一个组合全部子索引、读写串行化、支持快照读的统一入口，以便搜索即时返回且后台实时更新互不干扰。

**验收标准（Given / When / Then）**

- **AC1 — 组合索引**
  - **Given** InMemoryIndex 实例
  - **When** 查询
  - **Then** 合并 Trie + FullSubstringMap + TrigramIndex + PinyinIndex 并去重

- **AC2 — actor 隔离**
  - **Given** 并发访问
  - **When** 多任务读写
  - **Then** 所有读写经 actor isolation 串行化

- **AC3 — 快照读**
  - **Given** 正在进行的查询持有快照
  - **When** 并发插入 / 删除
  - **Then** 快照隔离，查询不受影响；`snapshot() -> IndexSnapshot` 返回不可变视图

- **AC4 — 构建性能**
  - **Given** 100 万 FileRecord
  - **When** 构建索引
  - **Then** 耗时 < 10 s

- **AC5 — 内存上限**
  - **Given** 100 万记录（平均 20 字符文件名）
  - **When** 估算内存
  - **Then** < 10 GB；超出时长名降级到 TrigramIndex

- **AC6 — 背压**
  - **Given** actor 队列
  - **When** pending mutations 超过 10 000
  - **Then** 丢弃最旧的 pending 并记录 warning（含丢弃数量与队列深度）

- **AC7 — 批量 API**
  - **Given** 批量插入 / 删除
  - **When** 调用 `insertBatch`/`deleteBatch`
  - **Then** 单次 actor hop 内完成，减少单条 mutation 开销

**实现**：`Sources/Index/InMemoryIndex.swift` · **测试**：`Tests/IndexTests/InMemoryIndexTests.swift`
