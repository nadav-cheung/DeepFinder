### REQ-0.7-01 daemon 子命令 ✅ 🖥️ P0

**用户故事**

作为「用户」，我希望通过 `deepfinder daemon` 子命令管理 daemon 的生命周期，以便无需手动 kill 进程即可启动、停止、重启并查看运行状态。

**验收标准（Given / When / Then）**

- **AC1 — start 启动 daemon**
  - **Given** daemon 未运行
  - **When** 执行 `deepfinder daemon start`
  - **Then** 启动 daemon 进程

- **AC2 — start 幂等提示**
  - **Given** daemon 已在运行（PID XXX）
  - **When** 执行 `deepfinder daemon start`
  - **Then** 友好提示 "daemon already running (PID XXX)"

- **AC3 — stop 优雅退出**
  - **Given** daemon 正在运行
  - **When** 执行 `deepfinder daemon stop`
  - **Then** 发送 SIGTERM 并等待退出（最多 5s）

- **AC4 — stop 超时提示**
  - **Given** daemon 在 5s 内未响应 SIGTERM
  - **When** 等待超时
  - **Then** 提示 "Daemon did not stop within 5 seconds. Try: kill -9 <PID>"

- **AC5 — restart**
  - **Given** daemon 运行中
  - **When** 执行 `deepfinder daemon restart`
  - **Then** 执行 stop + start，并确保 socket 文件清理后再启动

- **AC6 — status 运行态**
  - **Given** daemon 运行中
  - **When** 执行 `deepfinder daemon status`
  - **Then** 显示 PID、运行时长、索引状态、索引文件数、内存占用（MB）

- **AC7 — status 未运行**
  - **Given** daemon 未运行
  - **When** 执行 `deepfinder daemon status`
  - **Then** 显示 "daemon 未运行"

**实现**：`Sources/CLI/DaemonCommands.swift` · **测试**：`Tests/CLITests/DaemonCommandsTests.swift`

### REQ-0.7-02 config 子命令 ✅ 🖥️ P1

**用户故事**

作为「用户」，我希望通过 CLI 子命令管理配置，以便无需手动编辑 JSON 文件即可查看与修改配置项。

**验收标准（Given / When / Then）**

- **AC1 — get**
  - **Given** 任意配置键 KEY
  - **When** 执行 `deepfinder config get KEY`
  - **Then** 显示指定配置值

- **AC2 — set**
  - **Given** 配置键 KEY 与值 VALUE
  - **When** 执行 `deepfinder config set KEY VALUE`
  - **Then** 通过 IPC 写入 daemon，更新配置值

- **AC3 — list**
  - **Given** 配置存储
  - **When** 执行 `deepfinder config list`
  - **Then** 列出所有配置项及当前值

- **AC4 — reset**
  - **Given** 用户确认
  - **When** 执行 `deepfinder config reset`
  - **Then** 在确认提示后恢复默认配置

- **AC5 — 持久化与回读**
  - **Given** 配置已通过 set 变更
  - **When** daemon 接收变更
  - **Then** daemon 将新值原子写入 `settings.json` 并在后续 `config get` 回读中反映新值；运行中组件（扫描器/协调器）不热加载——重启后生效

**实现**：`Sources/CLI/ConfigCommands.swift` · **测试**：`Tests/CLITests/ConfigCommandsTests.swift`

### REQ-0.7-03 install 子命令 ✅ 🖥️ P1

**用户故事**

作为「用户」，我希望通过 `deepfinder install` 安装 LaunchAgent，以便开机自动启动 daemon；通过 `deepfinder uninstall` 移除之。

**验收标准（Given / When / Then）**

- **AC1 — install 写入 plist**
  - **Given** daemon 二进制可用
  - **When** 执行 `deepfinder install`
  - **Then** 安装 LaunchAgent plist 到 `~/Library/LaunchAgents/`，实现开机自启 daemon

- **AC2 — uninstall 移除 plist**
  - **Given** LaunchAgent 已安装
  - **When** 执行 `deepfinder uninstall`
  - **Then** 移除 LaunchAgent plist

- **AC3 — install 已安装提示**
  - **Given** LaunchAgent 已安装
  - **When** 执行 `deepfinder install`
  - **Then** 提示 "已安装，如需重新安装请先 uninstall"

- **AC4 — uninstall 未安装提示**
  - **Given** LaunchAgent 未安装
  - **When** 执行 `deepfinder uninstall`
  - **Then** 提示 "未安装"

- **AC5 — plist 内容**
  - **Given** 生成的 plist
  - **When** 检查字段
  - **Then** 包含 Label、ProgramArguments（指向 daemon 二进制）、Program、RunAtLoad=true、KeepAlive=true、StandardOutPath、StandardErrorPath

**实现**：`Sources/CLI/InstallCommands.swift` · **测试**：`Tests/CLITests/InstallCommandsTests.swift`
