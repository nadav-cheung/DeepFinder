# v2.0 — GUI + 扩展索引

18 项 REQ（P0: 9，P1: 7，P2: 2）

> 卡片格式（OSS · BDD）：用户故事 + 验收标准（Given/When/Then）+ 实现/测试溯源。金标准见 v0.1-index-core.md。

> **v2.0 在 v1.0 CLI Release 基础上增加 GUI 前端**。同一个 daemon 进程同时服务 CLI 和 GUI 客户端（通过 IPC）。GUI 通过 `x-apple.systempreferences:com.apple.preference.security?Privacy_AllFiles` 请求 Full Disk Access。所有 GUI REQs 通过 IPC 连接同一 daemon，不直接访问索引。

### REQ-2.0-01 SearchPanelView (NSPanel) ✅ 🖥️ P0

**用户故事**

作为「用户」，我希望按 ⌃⌘K 就有一个 Spotlight 风格的搜索面板从屏幕顶部滑出（半透明毛玻璃 + 光晕边框），以便第一印象就决定继续使用。

**验收标准（Given / When / Then）**

- **AC1 — 浮动层级窗口**
  - **Given** NSPanel 浮动窗口
  - **When** 构建面板
  - **Then** 使用 `.floating` 层级，无标题栏，无 Dock 图标

- **AC2 — Liquid Glass 材质**
  - **Given** 面板渲染
  - **When** 应用材质
  - **Then** 使用 `.glassEffect(.regular, in: .rect(cornerRadius: 24))`，并由 GlassEffectContainer 统一渲染

- **AC3 — 热键唤出**
  - **Given** 用户按下 ⌃⌘K
  - **When** 触发热键
  - **Then** 面板在当前活跃屏幕（非主屏幕）居中出现，搜索框自动获得焦点

- **AC4 — 关闭交互**
  - **Given** 面板已显示
  - **When** 点击面板外任意位置或按 Esc
  - **Then** 面板消失

- **AC5 — 文本保留**
  - **Given** 面板曾被关闭
  - **When** 再次按 ⌃⌘K 唤出
  - **Then** 上次的搜索文本和光标位置保留

- **AC6 — 屏幕跟随鼠标**
  - **Given** 多显示器环境
  - **When** 每次热键触发
  - **Then** 通过 NSScreen.mouseLocation 匹配，重新定位面板到鼠标所在屏幕

- **AC7 — 显示器断开兜底**
  - **Given** 面板所在显示器被断开
  - **When** 显示器移除
  - **Then** 面板自动迁移到主显示器

- **AC8 — 宽度约束**
  - **Given** 屏幕宽度
  - **When** 计算面板宽度
  - **Then** 不超过屏幕宽度减两侧边距，最小 480pt，最大 800pt

- **AC9 — main screen fallback**
  - **Given** `NSScreen.main` 返回 nil
  - **When** 解析屏幕
  - **Then** fallback 到 `NSScreen.screens[0]`

- **AC10 — reduceMotion 降级**
  - **Given** 系统启用 reduceMotion
  - **When** 显示/隐藏面板
  - **Then** 无动画，直接显示/隐藏

- **AC11 — 通过 IPC 访问 daemon**
  - **Given** GUI 面板
  - **When** 发起查询
  - **Then** 通过 IPC 连接 daemon，不直接访问索引

**实现**：`Sources/GUI/SearchPanelView.swift` · **测试**：`Tests/GUITests/SearchPanelTests.swift`

### REQ-2.0-02 SearchBarView ✅ 🖥️ P0

**用户故事**

作为「用户」，我希望看到搜索框直接打字就即时出现结果，无需点击搜索按钮或按 Enter，以便零阻力地搜索。

**验收标准（Given / When / Then）**

- **AC1 — 布局结构**
  - **Given** 搜索栏
  - **When** 渲染
  - **Then** 左侧搜索图标 + SwiftUI TextField（自定义样式）+ 右侧清除按钮；如需 NSTextField 特定行为（更精细的文本选择/委托），通过 NSViewRepresentable 桥接并记录原因

- **AC2 — Liquid Glass 材质**
  - **Given** 搜索栏
  - **When** 渲染
  - **Then** 使用 `.glassEffect()` 材质

- **AC3 — 即时搜索（CJK 感知）**
  - **Given** 搜索框聚焦且光标闪烁
  - **When** 输入文字
  - **Then** 即时触发搜索（无防抖），但 CJK 输入法组合期间不触发（监听 hasMarkedText，仅在文本提交后搜索）

- **AC4 — 清除按钮**
  - **Given** 搜索框内有文字
  - **When** 点击 ✕ 清除按钮
  - **Then** 清空搜索框与结果；清除按钮仅在有文字时显示

- **AC5 — 长文字滚动**
  - **Given** 输入超长文本
  - **When** 文本超出宽度
  - **Then** 搜索框可水平滚动

- **AC6 — Placeholder 与无障碍**
  - **Given** 空搜索框
  - **When** 渲染
  - **Then** Placeholder 灰显 "搜索文件..."；VoiceOver 读出 Accessibility label "搜索文件"

**实现**：`Sources/GUI/SearchBarView.swift` · **测试**：`Tests/GUITests/SearchBarTests.swift`

  - **When** 按下到面板出现

  - **When** 各自失败

- **AC8 — 瞬态失败重试**

  - **When** 重试
  - **Then** 最多 3 次，指数退避

## Standalone App Target

DeepFinder 有两个用户产品：(1) deepfinder CLI（程序员用，Homebrew formula 分发），(2) DeepFinder App（普通用户用，.app bundle 分发）。GUI 代码已实现（19 文件在 Sources/GUI/），但缺少独立 app 可执行目标。以下 REQs 定义如何将 GUI 代码打包为可分发的 macOS 应用。

---

### REQ-2.0-14 App 可执行目标 ✅ 🖥️ P0

**用户故事**

作为「开发者」，我希望运行 `scripts/build-app.sh` 就能生成 `DeepFinder.app`、双击即可运行（和任何 macOS 应用一样，不需要 Xcode 工程），以便交付普通用户。

**验收标准（Given / When / Then）**

- **AC1 — main 入口**
  - **Given** `Sources/AppEntry/main.swift`
  - **When** 构建
  - **Then** 包含 `@main struct DeepFinderAppEntry`

- **AC2 — application 配置**
  - **Given** main 入口
  - **When** 初始化
  - **Then** 创建 `NSApplication.shared`，设置 `.accessory` activation policy（无 Dock 图标）

- **AC3 — delegate 与配置**
  - **Given** main 入口
  - **When** 设置 delegate
  - **Then** 设置 `DeepFinderAppDelegate` 为 delegate，使用 `DeepFinderAppConfiguration.production()` 配置

- **AC4 — SPM 目标**
  - **Given** `Package.swift`
  - **When** 添加目标
  - **Then** 添加 `.executableTarget "DeepFinderApp"`，依赖 `DeepFinder` library

- **AC5 — 跨 target 可访问**
  - **Given** 跨 target 访问
  - **When** 引用
  - **Then** `DeepFinderAppDelegate` 和 `DeepFinderAppConfiguration` 标记为 `public`

**实现**：`Sources/AppEntry/main.swift` · **测试**：—

### REQ-2.0-15 App Bundle Info.plist ✅ 🖥️ P0

**用户故事**

作为「用户」，我希望 DeepFinder.app 在 macOS 上正确识别为菜单栏应用（无 Dock 图标、无 Cmd+Tab 条目、版本信息正确），以便体验符合预期。

**验收标准（Given / When / Then）**

- **AC1 — LSUIElement**
  - **Given** `App/Info.plist`
  - **When** 配置
  - **Then** 包含 `LSUIElement=true`（无 Dock 图标，无 Cmd+Tab）

- **AC2 — CFBundleIdentifier**
  - **Given** Info.plist
  - **When** 配置
  - **Then** `CFBundleIdentifier` 从 ProductConfig 获取（如 `cn.com.nadav.deepfinder`）

- **AC3 — CFBundleExecutable**
  - **Given** Info.plist
  - **When** 配置
  - **Then** `CFBundleExecutable` 匹配可执行目标名（`DeepFinderApp`）

- **AC4 — 版本号注入**
  - **Given** Info.plist
  - **When** 构建
  - **Then** 版本号通过 `{{VERSION}}` 占位符从 `VERSION` 文件注入

- **AC5 — NSPrincipalClass**
  - **Given** Info.plist
  - **When** 配置
  - **Then** `NSPrincipalClass = NSApplication`

- **AC6 — 显示名**
  - **Given** Info.plist
  - **When** 配置
  - **Then** `CFBundleName` 和 `CFBundleDisplayName` 使用产品显示名

**验收标准（Given / When / Then）**

  - **Given** `scripts/build-app.sh`
  - **When** 执行
  - **Then** 构建 SPM 目标，同时构建 `DeepFinderApp` 和 `DeepFinderDaemon` 两个可执行目标

  - **Given** Info.plist

  - **Then** 将 `VERSION` 文件内容注入 Info.plist 的 `{{VERSION}}` 占位符

- **AC4 — Ad-hoc 签名**
  - **Given** 构建输出
  - **When** 签名
  - **Then** Ad-hoc code signing（`codesign --sign -`）用于开发阶段

- **AC5 — 幂等**
  - **Given** 脚本可重复运行
  - **When** 多次执行
  - **Then** 输出到 `build/DeepFinder.app`，幂等无副作用
