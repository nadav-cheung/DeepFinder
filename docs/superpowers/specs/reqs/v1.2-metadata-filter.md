# v1.2 — 元数据过滤

8 项 REQ（P0: 7，P1: 1）

> 卡片格式（OSS · BDD）：用户故事 + 验收标准（Given/When/Then）+ 实现/测试溯源。金标准见 v0.1-index-core.md。

**设计原则**：过滤器作为搜索查询的修饰词，与 v1.1 高级搜索语法（布尔、通配符、正则）正交组合。过滤在 SearchCoordinator 层执行，先从索引获取文本匹配结果，再用 FilterExpression 管线过滤。

**前置依赖**：v1.1 高级搜索语法（SearchQuery 解析管线）。v1.2 的过滤器解析集成到同一管线中。

### REQ-1.2-01 Size 过滤器（size:>1mb, size:100kb..10mb）✅ 🖥️ P0

**用户故事**

作为「用户」，我希望在搜索查询中混入大小过滤（如 `size:>1gb`、`size:100kb..10mb`、`report size:<1mb`），以便在清理磁盘时快速找出特定大小的文件。

**验收标准（Given / When / Then）**

- **AC1 — 比较形式**
  - **Given** 文件大小已索引
  - **When** 输入 `size:><value>`、`size:<<value>`、`size:>=<value>`、`size:<=<value>`、`size:<value>`、`size:<start>..<end>` 任一形式
  - **Then** 返回符合对应比较语义的文件（范围语法包含两端）

- **AC2 — 单位解析**
  - **Given** 大小数值
  - **When** 带 `kb`/`mb`/`gb` 后缀（不区分大小写）或无后缀
  - **Then** 无后缀按字节计；kb=1024、mb=1024²、gb=1024³，解析为 Int64 字节值

- **AC3 — 零 I/O 内存比较**
  - **Given** 已索引的 FileRecord
  - **When** 应用大小过滤器
  - **Then** 基于 FileRecord.size (Int64) 直接在内存中比较，不触发文件系统 I/O

- **AC4 — 文件夹处理**
  - **Given** 目录的 size 字段为 0
  - **When** 大小过滤
  - **Then** 直接按 size 字段比较；目录因 size==0 命中 `size:<N`/`size:0`，需配合 `folder:` 显式限定

- **AC5 — 无效输入**
  - **Given** 无效 size 语法（如 `size:abc`）
  - **When** 解析
  - **Then** 返回 nil，该过滤器被静默跳过（不产生错误信息）

- **AC6 — 与文本搜索组合**
  - **Given** 名称已索引的文件
  - **When** 输入 `report size:<1mb`
  - **Then** 返回名称含 "report" 且小于 1MB 的文件

- **AC7 — 测试覆盖**
  - **Given** SearchFilterTests / FilterPipelineTests
  - **When** 运行测试套件
  - **Then** 覆盖所有比较运算符（>=、>、<=、<、bare）、所有单位（b/kb/mb/gb）、范围语法、无效输入返回 nil

**实现**：`Sources/Search/SearchFilter.swift`、`Sources/Search/FilterPipeline.swift` · **测试**：`Tests/SearchTests/SearchFilterTests.swift`、`Tests/SearchTests/FilterPipelineTests.swift`

### REQ-1.2-02 日期过滤器（dm:today, dc:thisweek, dm:2026-01-01..2026-03-31）✅ 🖥️ P0

**用户故事**

作为「用户」，我希望按修改/创建日期过滤文件（如 `dm:today`、`dc:thisweek`、`report dm:2026-01-01..2026-03-31`），以便快速定位近期或特定时间段内的文件。

**验收标准（Given / When / Then）**

- **AC1 — 双前缀**
  - **Given** 两个日期前缀
  - **When** 使用 `dm:` 或 `dc:`
  - **Then** 分别按 FileRecord.modifiedAt / FileRecord.createdAt (Date) 过滤，两者支持完全相同的日期语法

- **AC2 — 比较运算符**
  - **Given** 创建日期字段（dc: 前缀）
  - **When** 输入 `dc:>=<date>`、`dc:>><date>`、`dc:<=<date>`、`dc:<<date>`
  - **Then** 按对应比较语义返回结果（dm: 前缀不支持比较运算符前缀，仅支持常量与范围）

- **AC3 — 日期格式**
  - **Given** ISO 日期格式
  - **When** 输入 `YYYY-MM-DD`（经 `Product.isoDateFormat` / DateFormatter 解析）
  - **Then** 正确解析

- **AC4 — 日期常量**
  - **Given** 5 个常量
  - **When** 输入 `today`/`yesterday`/`thisweek`/`thismonth`/`thisyear`
  - **Then** 按经 Foundation `Calendar`（gregorian）API 计算对应区间

- **AC5 — 范围语法**
  - **Given** 显式日期范围语法
  - **When** 输入 `dm:2026-01-01..2026-03-31`
  - **Then** 返回 dateModifiedRange，下界为 startOfDay、上界排他（next day startOfDay，即包含 3 月 31 日全天）

- **AC6 — 无效输入**
  - **Given** 无效日期语法
  - **When** 解析
  - **Then** 返回 nil，该过滤器被静默跳过（不产生错误信息）

- **AC7 — 测试覆盖**
  - **Given** SearchFilterTests / FilterPipelineTests
  - **When** 运行测试套件
  - **Then** 覆盖各日期常量、范围语法、ISO 日期格式、dc: 比较运算符、无效输入

**实现**：`Sources/Search/SearchFilter.swift`、`Sources/Search/FilterPipeline.swift` · **测试**：`Tests/SearchTests/SearchFilterTests.swift`、`Tests/SearchTests/FilterPipelineTests.swift`

### REQ-1.2-03 扩展名过滤器（ext:pdf;doc;xlsx）✅ 🖥️ P0

**用户故事**

作为「用户」，我希望按文件扩展名过滤（如 `report ext:pdf;doc;xlsx`），以便一次限定到多种文件类型，无需记住复杂扩展名清单。

**验收标准（Given / When / Then）**

- **AC1 — 单/多扩展名**
  - **Given** 扩展名语法
  - **When** 输入 `ext:pdf` 或 `ext:pdf;doc;xlsx`
  - **Then** 单扩展名匹配该类型；分号分隔的多扩展名为 OR 关系，返回任一匹配

- **AC2 — 大小写不敏感**
  - **Given** FileRecord.extension 为 NFC + lowercased
  - **When** 输入 `ext:PDF`
  - **Then** 等同于 `ext:pdf`，正确匹配

- **AC3 — 空扩展名**
  - **Given** 无扩展名的文件
  - **When** 输入 `ext:`（空值）
  - **Then** 匹配 FileRecord.extension == nil 的文件

- **AC4 — 不含前导点**
  - **Given** 扩展名规范
  - **When** 输入 `ext:pdf`
  - **Then** 正确匹配；`ext:.pdf`（带前导点）不符合规范

- **AC5 — 零 I/O**
  - **Given** 已索引的 FileRecord
  - **When** 应用扩展名过滤器
  - **Then** 基于 FileRecord.extension 字段直接比较，不涉及文件系统 I/O

- **AC6 — 重复过滤器**
  - **Given** 同一查询
  - **When** 输入 `ext:pdf ext:doc`
  - **Then** 两个 extensionFilter 均加入管线，按 AND 语义叠加（实际返回交集），不报错

- **AC7 — 文件夹匹配**
  - **Given** 文件夹（FileRecord.isDirectory == true）
  - **When** extension 字段可能为 nil 或有值（如 ".app"）
  - **Then** 按实际 extension 字段匹配

- **AC8 — 无效输入**
  - **Given** 无效字符或语法
  - **When** 解析
  - **Then** 无法构成有效扩展名集合时返回 nil，该过滤器被静默跳过（不产生错误信息）

- **AC9 — 测试覆盖**
  - **Given** SearchFilterTests / FilterPipelineTests
  - **When** 运行测试套件
  - **Then** 覆盖单扩展名、多扩展名、空扩展名、大小写混合

**实现**：`Sources/Search/SearchFilter.swift`、`Sources/Search/FilterPipeline.swift` · **测试**：`Tests/SearchTests/SearchFilterTests.swift`、`Tests/SearchTests/FilterPipelineTests.swift`

### REQ-1.2-04 文件类型宏（audio:, video:, pic:, doc:）✅ 🖥️ P0

**用户故事**

作为「用户」，我希望用类型宏（`video:`、`pic:`、`audio:`、`doc:`）一键过滤常见媒体/文档类型，以便无需手动列举大量扩展名（如 `ext:mp4;avi;mkv;mov;...`）。

**验收标准（Given / When / Then）**

- **AC1 — 四个内置宏**
  - **Given** 对标 Everything 的 4 个宏
  - **When** 输入 `audio:`/`video:`/`pic:`/`doc:`
  - **Then** 分别匹配音频/视频/图片/文档文件

- **AC2 — 宏展开**
  - **Given** 宏定义（`FileTypeGroup` enum）
  - **When** 展开
  - **Then** 每个宏映射到内置扩展名集合（硬编码，lowercased）：audio=mp3/wav/aac/flac/ogg/wma/m4a；video=mp4/mov/avi/mkv/wmv/flv/webm；picture=jpg/jpeg/png/gif/bmp/svg/webp/tiff/ico；document=pdf/doc/docx/xls/xlsx/ppt/pptx/txt/rtf/odt

- **AC3 — 大小写不敏感**
  - **Given** 与 ext: 行为一致
  - **When** 匹配
  - **Then** 扩展名不区分大小写

- **AC4 — 与文本搜索组合**
  - **Given** 名称已索引的文件
  - **When** 输入 `video: movie` 或 `pic: sunset`
  - **Then** 视频文件 AND 名称含 "movie"（或图片文件 AND 名称含 "sunset"）

- **AC5 — 与 ext: 同时使用**
  - **Given** 宏与扩展名过滤器
  - **When** 同一查询中同时使用
  - **Then** AND 语义：结果必须同时满足两个过滤器

- **AC6 — 未知宏**
  - **Given** 未定义的宏
  - **When** 输入 `code:`
  - **Then** 该修饰词被静默忽略（落入 default 分支，不产生过滤器也不报错）

- **AC7 — 可测试性**
  - **Given** FileTypeGroup enum
  - **When** 通过 `FileTypeGroup.extensions` 公开访问
  - **Then** 扩展名集合可测试（硬编码，不依赖 UTI 查询，避免 I/O 确保确定性）

- **AC8 — 测试覆盖**
  - **Given** SearchFilterTests / FilterPipelineTests
  - **When** 运行测试套件
  - **Then** 覆盖每个宏的正确展开、与文本搜索组合、与 ext: 同时使用

**实现**：`Sources/Search/SearchFilter.swift`、`Sources/Search/FilterPipeline.swift` · **测试**：`Tests/SearchTests/SearchFilterTests.swift`、`Tests/SearchTests/FilterPipelineTests.swift`

### REQ-1.2-05 文件/文件夹过滤器（file:, folder:）✅ 🖥️ P0

**用户故事**

作为「用户」，我希望用 `file:` 或 `folder:` 限定结果只包含文件或只包含文件夹，以便快速排除不想要的目录项或文件项。

**验收标准（Given / When / Then）**

- **AC1 — file: 过滤**
  - **Given** 已索引的 FileRecord
  - **When** 输入 `file:` 或 `file: report`
  - **Then** 只返回 FileRecord.isDirectory == false 的记录

- **AC2 — folder: 过滤**
  - **Given** 已索引的 FileRecord
  - **When** 输入 `folder:` 或 `folder: project`
  - **Then** 只返回 FileRecord.isDirectory == true 的记录

- **AC3 — 同时使用**
  - **Given** 同一查询
  - **When** 同时使用 `file:` 和 `folder:`
  - **Then** 两个过滤器（isFile、isDirectory）均加入管线，按 AND 语义叠加；因一个记录不可能同时满足，结果为空（不报错）

- **AC4 — 单独使用**
  - **Given** 无文本搜索
  - **When** 输入 `file:` 或 `folder:`
  - **Then** 返回所有文件或所有文件夹

- **AC5 — 查询文本驱动**
  - **Given** CLI 无 `--type` flag（v1.2 过滤完全由查询文本中的 `file:`/`folder:` 表达）
  - **When** 用户输入 `report folder:` 或 `file: report`
  - **Then** file:/folder: 修饰词直接在查询文本中表达，经 QueryParser 抽取为 modifierPairs 后转入 FilterPipeline

- **AC6 — 测试覆盖**
  - **Given** SearchFilterTests / FilterPipelineTests
  - **When** 运行测试套件
  - **Then** 覆盖 file-only、folder-only、互斥错误、与文本搜索组合、单独使用

**实现**：`Sources/Search/SearchFilter.swift`、`Sources/Search/FilterPipeline.swift` · **测试**：`Tests/SearchTests/SearchFilterTests.swift`、`Tests/SearchTests/FilterPipelineTests.swift`

### REQ-1.2-06 路径深度过滤器（depth:3）✅ 🖥️ P0

**用户故事**

作为「用户」，我希望按路径深度过滤（如 `depth:3 report`），以便避免搜索到深层嵌套的 `node_modules` 或 `.build` 目录，让结果更精准。

**验收标准（Given / When / Then）**

- **AC1 — 比较运算符**
  - **Given** 路径深度可计算
  - **When** 输入 `depth:<=N`、`depth:<<N`、`depth:>=N`、`depth:>N`
  - **Then** 按对应比较语义返回结果（`<` 映射为 maxDepth(N-1)、`>` 映射为 minDepth(N+1)）

- **AC2 — 默认运算符**
  - **Given** 无运算符的语法
  - **When** 输入 `depth:3`
  - **Then** 默认 `<=N` 语义（对标 Everything），返回路径深度 <= 3 的项

- **AC3 — 深度计算**
  - **Given** FileRecord.path（符号链接解析后的实际路径）
  - **When** 计算 `path.components(separatedBy: "/").filter { !$0.isEmpty }.count`
  - **Then** `/`=0、`/Users/`=1、`/Users/nadav/`=2、`/Users/nadav/Documents/`=3、`/Users/nadav/Documents/report.pdf`=3

- **AC4 — 零 I/O**
  - **Given** 已索引的 FileRecord
  - **When** 应用深度过滤器
  - **Then** 基于 FileRecord.path 计算，不涉及文件系统 I/O

- **AC5 — 边界 N=0**
  - **Given** N 为非负整数
  - **When** 输入 `depth:0`
  - **Then** 只返回根路径项

- **AC6 — 无效输入**
  - **Given** 非整数或负数
  - **When** 输入 `depth:abc` 或 `depth:-1`
  - **Then** 返回 nil，该过滤器被静默跳过（不产生错误信息）

- **AC7 — 测试覆盖**
  - **Given** SearchFilterTests / FilterPipelineTests
  - **When** 运行测试套件
  - **Then** 覆盖各比较运算符（含 `<`/`>` 边界偏移）、bare 默认 `<=N`、边界值（0、1）、深层路径

**实现**：`Sources/Search/SearchFilter.swift`、`Sources/Search/FilterPipeline.swift` · **测试**：`Tests/SearchTests/SearchFilterTests.swift`、`Tests/SearchTests/FilterPipelineTests.swift`

### REQ-1.2-07 FilterExpression 模型 ✅ 🖥️ P1

**用户故事**

作为「系统」，我希望有一个统一的过滤器模型（FilterExpression），组合所有过滤器类型、可序列化、可测试，以便作为 v1.2 所有其他 REQ 的基础设施，确保过滤器之间正确组合与冲突检测。

**验收标准（Given / When / Then）**

- **AC1 — 模型完整**
  - **Given** SearchFilter enum（含 size/date/extension/fileType/depth/metadata/nameLength/caseSensitive 各 case）+ FileTypeGroup enum
  - **When** 编译
  - **Then** 全部声明为 Sendable + Equatable

- **AC2 — matches 方法**
  - **Given** SearchFilter 各 case
  - **When** 调用 `SearchFilter.matches(_:) -> Bool`
  - **Then** 对 FileRecord 应用对应比较；FilterPipeline.apply(to:) 对候选结果集施加所有过滤器

- **AC3 — AND 语义**
  - **Given** FilterPipeline.filters 中的多个过滤器
  - **When** 应用（如 `size:>1mb ext:pdf dm:today`）
  - **Then** 所有过滤器之间为 AND 关系，全部通过才返回 true

- **AC4 — 同类过滤器叠加**
  - **Given** 同类过滤器重复场景
  - **When** `file:` + `folder:` 共存，或 `ext:` 重复出现
  - **Then** 不报错；多个过滤器按 AND 叠加（`file:`+`folder:` 结果为空，多个 `ext:` 取交集）

- **AC5 — 不同类自由组合**
  - **Given** 不同类型的过滤器
  - **When** 组合（如 `size:>1mb ext:pdf dm:today`）
  - **Then** 自由组合，语义为各过滤器 AND

- **AC6 — Sendable**
  - **Given** FilterPipeline / SearchFilter / FileTypeGroup 模型
  - **When** 并发传递
  - **Then** 声明为 Sendable，可跨 actor 边界安全传递

- **AC7 — 空表达式**
  - **Given** 无过滤器的 FilterPipeline（filters == []）
  - **When** apply(to:) 任意结果集
  - **Then** 直接返回原结果集（匹配所有）

- **AC8 — 测试覆盖**
  - **Given** FilterPipelineTests
  - **When** 运行测试套件
  - **Then** 覆盖每个过滤器的 matches 方法、组合 AND 语义、空过滤器

**实现**：`Sources/Search/FilterPipeline.swift` · **测试**：`Tests/SearchTests/FilterPipelineTests.swift`

### REQ-1.2-08 FilterExpression 与 SearchCoordinator 集成 ✅ 🖥️ P0

**用户故事**

作为「用户」，我希望在 CLI 中输入 `report size:>1mb ext:pdf dm:today` 一次性得到名称含 "report" 且大于 1MB、PDF 格式、今天修改的结果，整个过滤过程在内存中完成、无文件系统 I/O。

**验收标准（Given / When / Then）**

- **AC1 — filters 参数**
  - **Given** SearchCoordinator.search 签名
  - **When** v1.2 集成
  - **Then** `search(query:filters:)` 接受 `[SearchFilter]`（默认空数组）；过滤器与查询文本分离传递

- **AC2 — 集成流程**
  - **Given** 用户查询文本 `report size:>1mb ext:pdf dm:today`
  - **When** 经 IPCServer → QueryParser 解析后 SearchCoordinator.search 执行
  - **Then** QueryParser 抽取 modifierPairs 并剥离为 textOnlyQuery；先由 SearchProvider.search 得到文本匹配候选结果，再由 FilterPipeline 过滤，最后排序返回

- **AC3 — 解析管线集成**
  - **Given** v1.1 QueryParser 解析管线
  - **When** 扩展
  - **Then** 过滤器解析集成到同一管线：QueryParser.parse() 输出 modifierPairs + textOnlyQuery；FilterPipeline.parse(from:) 转为过滤器数组

- **AC4 — 无效值静默跳过**
  - **Given** 过滤器无效值（size:abc, depth:-1）
  - **When** 解析
  - **Then** 对应 parseSizeFilter/parseDepthFilter 返回 nil，该修饰词被静默跳过，不报错，搜索继续执行

- **AC5 — 性能**
  - **Given** 所有过滤器字段来自 FileRecord 已有字段（size/createdAt/modifiedAt/extension/isDirectory/path）
  - **When** 单条记录过滤评估
  - **Then** 过滤器在 SearchCoordinator 内执行（FilterPipeline.apply），SearchProvider 层无感知；性能基准仅测量报告，不设硬阈值（手动目标：< 1μs/记录，10 万条 < 100ms）

- **AC6 — IPC 协议**
  - **Given** IPCRequest.query
  - **When** v1.2
  - **Then** query 请求仅携带 `(query: String, limit: Int?)`；过滤器从 `query` 文本在 daemon 侧解析（向后兼容，无独立 filterExpression 字段）

- **AC7 — REPL 集成**
  - **Given** REPL 交互环境
  - **When** 输入过滤器查询
  - **Then** 搜索命令支持完整过滤器语法（如 `> report size:>1mb ext:pdf`）；Tab 补全由 CompletionContext 提供（含命令/修饰词前缀补全）

- **AC8 — 测试覆盖**
  - **Given** FilterPipelineTests / SearchFilterTests
  - **When** 运行测试套件
  - **Then** 覆盖过滤器 + 文本搜索组合、过滤器 matches 评估、各修饰词解析（含 size/ext/dm/dc/depth/audio/video/pic/doc/file/folder）

**实现**：`Sources/Search/SearchCoordinator.swift`、`Sources/Search/FilterPipeline.swift` · **测试**：`Tests/SearchTests/SearchCoordinatorTests.swift`、`Tests/SearchTests/FilterPipelineTests.swift`
