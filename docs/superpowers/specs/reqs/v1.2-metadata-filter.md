# v1.2 — 元数据过滤

8 项 REQ（P0: 7，P1: 1）

> 卡片格式（OSS · BDD）：用户故事 + 验收标准（Given/When/Then）+ 实现/测试溯源。金标准见 v0.1-index-core.md。

**设计原则**：过滤器作为搜索查询的修饰词，与 v1.1 高级搜索语法（布尔、通配符、正则）正交组合。过滤在 SearchCoordinator 层执行，先从索引获取文本匹配结果，再用 FilterExpression 管线过滤。

**前置依赖**：v1.1 高级搜索语法（SearchQuery 解析管线）。v1.2 的过滤器解析集成到同一管线中。

### REQ-1.2-01 Size 过滤器（size:>1mb, size:100kb..10mb）✅ 🖥️ P0

**用户故事**

作为「用户」，我希望在搜索查询中混入大小过滤（如 `size:>1gb`、`size:100kb..10mb`、`report size:<1mb`），以便在清理磁盘时快速找出特定大小的文件。

**验收标准（Given / When / Then）**

- **AC1 — 比较形式**
  - **Given** 文件大小已索引
  - **When** 输入 `size:><value>`、`size:<<value>`、`size:>=<value>`、`size:<=<value>`、`size:<value>`、`size:<start>..<end>` 任一形式
  - **Then** 返回符合对应比较语义的文件（范围语法包含两端）

- **AC2 — 单位解析**
  - **Given** 大小数值
  - **When** 带 `kb`/`mb`/`gb` 后缀（不区分大小写）或无后缀
  - **Then** 无后缀按字节计；kb=1024、mb=1024²、gb=1024³，解析为 Int64 字节值

- **AC3 — 大小常量**
  - **Given** 对标 Everything 的 7 个常量
  - **When** 输入 `size:empty`/`size:tiny`/`size:small`/`size:medium`/`size:large`/`size:huge`/`size:gigantic`
  - **Then** 分别匹配 size==0、0<size≤10KB、10KB<size≤100KB、100KB<size≤1MB、1MB<size≤16MB、16MB<size≤128MB、size>128MB

- **AC4 — 零 I/O 内存比较**
  - **Given** 已索引的 FileRecord
  - **When** 应用大小过滤器
  - **Then** 基于 FileRecord.size (Int64) 直接在内存中比较，不触发文件系统 I/O

- **AC5 — 文件夹处理**
  - **Given** 目录的 size 字段为 0
  - **When** 大小过滤
  - **Then** 目录默认不参与大小过滤，除非用户显式搜索 `folder: size:empty`

- **AC6 — 语法错误**
  - **Given** 无效 size 语法（如 `size:abc`）
  - **When** 解析
  - **Then** 返回明确错误信息：`"无效的 size 语法: '<input>'。用法: size:>10mb 或 size:100kb..1gb"`

- **AC7 — 与文本搜索组合**
  - **Given** 名称已索引的文件
  - **When** 输入 `report size:<1mb`
  - **Then** 返回名称含 "report" 且小于 1MB 的文件

- **AC8 — 测试覆盖**
  - **Given** SearchFilterTests / FilterPipelineTests
  - **When** 运行测试套件
  - **Then** 覆盖所有比较运算符、所有单位、每个常量、范围语法、边界值（0、1 字节、Int64.max 附近）、无效语法错误

**实现**：`Sources/Search/SearchFilter.swift`、`Sources/Search/FilterPipeline.swift` · **测试**：`Tests/SearchTests/SearchFilterTests.swift`、`Tests/SearchTests/FilterPipelineTests.swift`

### REQ-1.2-02 日期过滤器（dm:today, dc:thisweek, dm:2026-01-01..2026-03-31）✅ 🖥️ P0

**用户故事**

作为「用户」，我希望按修改/创建日期过滤文件（如 `dm:today`、`dc:thisweek`、`report dm:2026-01-01..2026-03-31`），以便快速定位近期或特定时间段内的文件。

**验收标准（Given / When / Then）**

- **AC1 — 双前缀**
  - **Given** 两个日期前缀
  - **When** 使用 `dm:` 或 `dc:`
  - **Then** 分别按 FileRecord.modifiedAt / FileRecord.createdAt (Date) 过滤，两者支持完全相同的日期语法

- **AC2 — 比较运算符**
  - **Given** 日期字段
  - **When** 输入 `dm:<date>`（精确）、`dm:<=<date>`、`dm:<<date>`、`dm:>=<date>`、`dm:>><date>`、`dm:<start>..<end>`（范围包含两端）
  - **Then** 按对应比较语义返回结果

- **AC3 — 日期格式**
  - **Given** 4 种日期格式
  - **When** 输入 `YYYY-MM-DD`、`YYYYMMDD`、`YYYY-MM-DDThh:mm`、`YYYY-MM-DDThh:mm:ss`
  - **Then** 全部正确解析

- **AC4 — 日期常量**
  - **Given** 对标 Everything 的 8 个常量
  - **When** 输入 `today`/`yesterday`/`thisweek`/`lastweek`/`thismonth`/`lastmonth`/`thisyear`/`lastyear`
  - **Then** 按本地时区（`TimeZone.current`）经 Foundation `Calendar` API 计算对应区间

- **AC5 — 精确匹配整天**
  - **Given** 无运算符的精确匹配语法
  - **When** 输入 `dm:2026-05-29`
  - **Then** 匹配当天 00:00:00 至 23:59:59 的全天范围（startOfDay ... endOfDay）

- **AC6 — 范围常量边界**
  - **Given** 范围常量
  - **When** 计算 `today`/`thisweek`/`thismonth` 或范围语法 `dm:2026-01-01..2026-03-31`
  - **Then** 起点取 startOfDay、终点取 endOfDay（thisweek 周一开始；范围语法包含 3 月 31 日全天）

- **AC7 — 语法错误**
  - **Given** 无效日期语法
  - **When** 解析
  - **Then** 返回明确错误信息：`"无效的日期语法: '<input>'。用法: dm:today 或 dm:2026-01-01..2026-03-31"`

- **AC8 — 测试覆盖**
  - **Given** SearchFilterTests / FilterPipelineTests
  - **When** 运行测试套件
  - **Then** 覆盖所有日期常量、所有比较运算符、范围语法、各日期格式、跨时区边界（mock 时区）、无效输入

**实现**：`Sources/Search/SearchFilter.swift`、`Sources/Search/FilterPipeline.swift` · **测试**：`Tests/SearchTests/SearchFilterTests.swift`、`Tests/SearchTests/FilterPipelineTests.swift`

### REQ-1.2-03 扩展名过滤器（ext:pdf;doc;xlsx）✅ 🖥️ P0

**用户故事**

作为「用户」，我希望按文件扩展名过滤（如 `report ext:pdf;doc;xlsx`），以便一次限定到多种文件类型，无需记住复杂扩展名清单。

**验收标准（Given / When / Then）**

- **AC1 — 单/多扩展名**
  - **Given** 扩展名语法
  - **When** 输入 `ext:pdf` 或 `ext:pdf;doc;xlsx`
  - **Then** 单扩展名匹配该类型；分号分隔的多扩展名为 OR 关系，返回任一匹配

- **AC2 — 大小写不敏感**
  - **Given** FileRecord.extension 为 NFC + lowercased
  - **When** 输入 `ext:PDF`
  - **Then** 等同于 `ext:pdf`，正确匹配

- **AC3 — 空扩展名**
  - **Given** 无扩展名的文件
  - **When** 输入 `ext:`（空值）
  - **Then** 匹配 FileRecord.extension == nil 的文件

- **AC4 — 不含前导点**
  - **Given** 扩展名规范
  - **When** 输入 `ext:pdf`
  - **Then** 正确匹配；`ext:.pdf`（带前导点）不符合规范

- **AC5 — 零 I/O**
  - **Given** 已索引的 FileRecord
  - **When** 应用扩展名过滤器
  - **Then** 基于 FileRecord.extension 字段直接比较，不涉及文件系统 I/O

- **AC6 — 重复过滤器报错**
  - **Given** 同一查询
  - **When** 输入 `ext:pdf ext:doc`
  - **Then** 报错（不静默覆盖）：`"ext: 过滤器只能出现一次，如需多扩展名请用分号: ext:pdf;doc"`

- **AC7 — 文件夹匹配**
  - **Given** 文件夹（FileRecord.isDirectory == true）
  - **When** extension 字段可能为 nil 或有值（如 ".app"）
  - **Then** 按实际 extension 字段匹配

- **AC8 — 语法错误**
  - **Given** 无效字符或语法
  - **When** 解析
  - **Then** 返回明确错误信息

- **AC9 — 测试覆盖**
  - **Given** SearchFilterTests / FilterPipelineTests
  - **When** 运行测试套件
  - **Then** 覆盖单扩展名、多扩展名、空扩展名、大小写混合、重复过滤器错误、无效字符

**实现**：`Sources/Search/SearchFilter.swift`、`Sources/Search/FilterPipeline.swift` · **测试**：`Tests/SearchTests/SearchFilterTests.swift`、`Tests/SearchTests/FilterPipelineTests.swift`

作为「用户」，我希望用类型宏（`video:`、`pic:`、`audio:`、`doc:`）一键过滤常见媒体/文档类型，以便无需手动列举大量扩展名（如 `ext:mp4;avi;mkv;mov;...`）。

  - **Then** 扩展名列表可测试（硬编码，不依赖 UTI 查询，避免 I/O 确保确定性）

- **AC8 — 测试覆盖**

### REQ-1.2-06 路径深度过滤器（depth:3）✅ 🖥️ P0

**用户故事**

作为「用户」，我希望按路径深度过滤（如 `depth:3 report`），以便避免搜索到深层嵌套的 `node_modules` 或 `.build` 目录，让结果更精准。

- **AC1 — 比较运算符**

### REQ-1.2-07 FilterExpression 模型 ✅ 🖥️ P1

**用户故事**

作为「系统」，我希望有一个统一的过滤器模型（FilterExpression），组合所有过滤器类型、可序列化、可测试，以便作为 v1.2 所有其他 REQ 的基础设施，确保过滤器之间正确组合与冲突检测。

**验收标准（Given / When / Then）**

- **AC1 — 模型完整**
  - **Given** Filter enum + 各子模型（SizeFilter / DateFilter / ExtensionFilter / FileTypeFilter / DepthFilter / TypeMacroFilter + ComparisonOperator）
  - **When** 编译
  - **Then** 全部声明为 Codable + Sendable + Equatable

- **AC2 — matches 方法**
  - **Given** FilterExpression 与各子模型
  - **When** 调用 `FilterExpression.matches(_:) -> Bool`
  - **Then** 对 FileRecord 应用所有过滤器；每个子模型也有独立的 `matches(_:) -> Bool`

- **AC3 — AND 语义**
  - **Given** FilterExpression.filters 中的多个过滤器
  - **When** 应用（如 `size:>1mb ext:pdf dm:today`）
  - **Then** 所有过滤器之间为 AND 关系，全部通过才返回 true

- **AC4 — 冲突检测**
  - **Given** 同类过滤器冲突场景
  - **When** `file:` + `folder:` 共存，或 `ext:` 重复出现
  - **Then** 报错

- **AC5 — 不同类自由组合**
  - **Given** 不同类型的过滤器
  - **When** 组合（如 `size:>1mb ext:pdf dm:today`）
  - **Then** 自由组合，语义为各过滤器 AND

- **AC6 — 可序列化**
  - **Given** FilterExpression 模型
  - **When** Codable 编解码
  - **Then** 支持 IPC 传输和持久化

- **AC7 — 空表达式**
  - **Given** 无过滤器的 FilterExpression
  - **When** matches 任意记录
  - **Then** 返回 true（匹配所有）

- **AC8 — 测试覆盖**
  - **Given** FilterPipelineTests
  - **When** 运行测试套件
  - **Then** 覆盖每个过滤器的 matches 方法、组合 AND 语义、冲突检测、空过滤器

**实现**：`Sources/Search/FilterPipeline.swift` · **测试**：`Tests/SearchTests/FilterPipelineTests.swift`

### REQ-1.2-08 FilterExpression 与 SearchCoordinator 集成 ✅ 🖥️ P0

**用户故事**

作为「用户」，我希望在 CLI 中输入 `report size:>1mb ext:pdf dm:today` 一次性得到名称含 "report" 且大于 1MB、PDF 格式、今天修改的结果，整个过滤过程在内存中完成、无文件系统 I/O。

**验收标准（Given / When / Then）**

- **AC1 — SearchQuery 扩展**
  - **Given** SearchQuery 结构
  - **When** v1.2 扩展
  - **Then** 新增 `filter: FilterExpression` 字段

- **AC2 — 集成流程**
  - **Given** 用户查询文本 `report size:>1mb ext:pdf dm:today`
  - **When** 经 QueryParser 解析后 SearchCoordinator.search 执行
  - **Then** 先由 SearchProvider.search 得到文本匹配候选结果，再由 FilterExpression 过滤，最后排序返回

- **AC3 — 解析管线集成**
  - **Given** v1.1 QueryParser 解析管线
  - **When** 扩展（或调用独立 FilterParser）
  - **Then** 过滤器解析集成到同一管线，输出 SearchQuery.text + FilterExpression.filters

- **AC4 — 错误在解析阶段报告**
  - **Given** 过滤器语法错误、冲突（file: + folder:）或无效值（size:abc, depth:-1）
  - **When** 解析
  - **Then** 解析阶段报错（格式 `"过滤器错误: <具体原因>。用法: <正确语法示例>"`），不执行搜索

- **AC5 — 性能**
  - **Given** 所有过滤器字段来自 FileRecord 已有字段（size/createdAt/modifiedAt/extension/isDirectory/path）
  - **When** 单条记录过滤评估
  - **Then** < 1μs（10 万条记录过滤 < 100ms）；过滤器在 SearchCoordinator 内执行，SearchProvider 层无感知，不影响结果排序

- **AC6 — IPC 扩展**
  - **Given** IPCQueryRequest
  - **When** v1.2 扩展
  - **Then** 新增 `filterExpression: Data?`（可选，向后兼容）；为 nil 时从 `text` 解析

- **AC7 — CLI flag 与查询文本优先级**
  - **Given** CLI flag `--type` 与查询文本过滤器共存
  - **When** 不同场景
  - **Then** 查询文本中的 `file:`/`folder:` 优先；`deepfinder "report size:>1mb" --type folder` 时 `size:` 生效、`--type folder` 生效（无 file:/folder: 冲突时）；`deepfinder "file: report" --type folder` 冲突报错

- **AC8 — REPL 集成**
