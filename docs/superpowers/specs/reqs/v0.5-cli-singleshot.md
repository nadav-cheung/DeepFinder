### REQ-0.5-01 CLIMain Single-Shot 模式 ✅ 🖥️ P0

**用户故事**

作为「用户」，我希望在终端输入 `deepfinder "report"` 后即时获得搜索结果并退出，以便像 `grep` 一样在脚本与命令行中得到即查即走的反馈。

**验收标准（Given / When / Then）**

- **AC1 — 单次查询退出**
  - **Given** 终端已可用、daemon 运行中
  - **When** 执行 `deepfinder "query"`
  - **Then** 完成一次搜索、输出结果后立即退出（不进入 REPL）

- **AC2 — 零依赖参数解析**
  - **Given** CLI 启动
  - **When** 解析参数
  - **Then** 手动处理 `CommandLine.arguments`，不引入任何外部依赖

- **AC3 — `--json` 结构化输出**
  - **Given** 需要将结果接入脚本管道
  - **When** 附加 `--json`
  - **Then** 输出结构化 JSON，适合脚本消费

- **AC4 — `--0` NUL 分隔输出**
  - **Given** 需要与 `xargs -0` 配合
  - **When** 附加 `--0`
  - **Then** 输出 NUL 分隔的纯路径

- **AC5 — `--sort` 排序**
  - **Given** 需要按指定维度排序
  - **When** 附加 `--sort name|size|date`
  - **Then** 按对应键排序结果

- **AC6 — `--limit` 限制条数**
  - **Given** 只需前 N 条
  - **When** 附加 `--limit N`
  - **Then** 最多输出 N 条

- **AC7 — `--reverse` 逆序**
  - **Given** 需要反向排列
  - **When** 附加 `--reverse`
  - **Then** 结果逆序输出

- **AC8 — `--verbose` 额外信息**
  - **Given** 需要更多匹配细节
  - **When** 附加 `--verbose`
  - **Then** 显示匹配类型、评分等额外信息

- **AC9 — IPC 全链路**
  - **Given** single-shot 调用
  - **When** 执行
  - **Then** 连接 daemon IPC → 发送查询 → 接收结果 → 格式化输出 → 退出

- **AC10 — Exit codes**
  - **Given** 不同执行结果
  - **When** 进程退出
  - **Then** 退出码 0=成功、1=无结果、2=daemon 错误、3=查询错误

**实现**：`Sources/CLI/CLIMain.swift`, `Sources/CLI/SingleShot.swift` · **测试**：`Tests/CLITests/CLIMainTests.swift`

### REQ-0.5-02 TerminalFormatter ✅ 🖥️ P0

**用户故事**

作为「用户」，我希望搜索结果以美观、易读的格式显示在终端——匹配部分高亮、文件信息一目了然，以便快速定位目标文件。

**验收标准（Given / When / Then）**

- **AC1 — 匹配子串高亮**
  - **Given** 一条命中的结果
  - **When** 渲染输出
  - **Then** 用 ANSI bold/color 突出显示匹配位置

- **AC2 — 保留原始大小写**
  - **Given** 查询「PORT」、文件名 `report.pdf`
  - **When** 渲染
  - **Then** 文件名保留原始大小写（`re**port**.pdf`）

- **AC3 — 路径缩短**
  - **Given** 路径位于 home 目录下
  - **When** 渲染路径
  - **Then** home 目录前缀替换为 `~/`

- **AC4 — 大小与日期灰显**
  - **Given** 文件大小与修改日期
  - **When** 渲染
  - **Then** 用 ANSI dim 灰显

- **AC5 — isatty 自动检测**
  - **Given** 输出被管道或重定向
  - **When** 检测到非 tty
  - **Then** 输出纯文本，不含 ANSI escape codes

- **AC6 — `--json` 模式**
  - **Given** 启用 `--json`
  - **When** 渲染
  - **Then** 输出结构化 JSON（SearchResult 数组）

- **AC7 — `--0` 模式**
  - **Given** 启用 `--0`
  - **When** 渲染
  - **Then** 输出 NUL 分隔的纯路径（无其他信息）

- **AC8 — 分页支持**
  - **Given** 大量结果
  - **When** 指定 `--limit N` + `--offset M`
  - **Then** 输出对应分页

- **AC9 — 拼音高亮中文**
  - **Given** 查询「bg」命中 `报告.pdf`
  - **When** 渲染
  - **Then** 高亮中文字符「**报告**」

- **AC10 — 多处匹配全高亮**
  - **Given** 查询「port」、文件名 `airport_transport_report.pdf` 含 3 处「port」
  - **When** 渲染
  - **Then** 3 处全部高亮

**实现**：`Sources/CLI/TerminalFormatter.swift` · **测试**：`Tests/CLITests/TerminalFormatterTests.swift`

### REQ-0.5-03 IPCClient ✅ 🖥️ P0

**用户故事**

作为「用户」，我希望无感知 IPC 通信细节——CLI 自动连接 daemon、发送查询、获得结果；连接失败时有清晰提示，以便无需了解底层机制即可使用。

**验收标准（Given / When / Then）**

- **AC1 — IPC 请求/响应**
  - **Given** CLI 启动、daemon 可达
  - **When** 发起查询
  - **Then** 连接 Unix domain socket，发送 IPCRequest，接收 IPCResponse

- **AC2 — 超时处理**
  - **Given** daemon 响应缓慢
  - **When** 连接或查询
  - **Then** 连接超时 5s、查询超时 10s

- **AC3 — Daemon 自动启动**
  - **Given** daemon 未运行
  - **When** CLI 启动
  - **Then** 自动 spawn daemon 进程 → 等待 ready 信号（最多 10s）→ 连接

- **AC4 — 重连逻辑**
  - **Given** daemon 崩溃
  - **When** 连接中断
  - **Then** 最多重试 3 次，间隔 1s

- **AC5 — 连接失败友好提示**
  - **Given** 所有重试均失败
  - **When** 终止连接尝试
  - **Then** 输出友好提示「无法连接 daemon，请运行 `deepfinder daemon start`」

**实现**：`Sources/Daemon/IPCClient.swift`, `Sources/CLI/IPCClientProtocol.swift` · **测试**：`Tests/CLITests/IntegrationTests.swift`

### REQ-0.5-04 CLI 参数解析 ✅ 🖥️ P0

**用户故事**

作为「用户」，我希望输入 `deepfinder --help` 能看到完整用法文档、输入错误参数能得到友好提示与示例，以便无需查阅外部资料即可学会使用。

**验收标准（Given / When / Then）**

- **AC1 — 零依赖手动解析**
  - **Given** CLI 启动
  - **When** 解析参数
  - **Then** 手动处理 `CommandLine.arguments`，不引入外部依赖

- **AC2 — 子命令路由**
  - **Given** 不同的入口形态
  - **When** 路由分发
  - **Then** `deepfinder [query]` → single-shot、`deepfinder`（无参数）→ REPL（v0.6）、`deepfinder daemon` → 子命令组（v0.7）

- **AC3 — Flag 解析**
  - **Given** 支持的 flag 集合
  - **When** 解析
  - **Then** 识别 `--json`, `--0`, `--sort`, `--limit`, `--reverse`, `--verbose`, `--help`, `--version`

- **AC4 — 错误处理**
  - **Given** 未知 flag
  - **When** 解析失败
  - **Then** 输出友好错误 + 用法提示，不 crash

- **AC5 — `--help` 用法文档**
  - **Given** 输入 `deepfinder --help`
  - **When** 触发
  - **Then** 输出完整用法文档（含示例）

- **AC6 — `--version` 版本号**
  - **Given** 输入 `deepfinder --version`
  - **When** 触发
  - **Then** 输出语义化版本号（读自 VERSION 文件）

**实现**：`Sources/CLI/ArgParser.swift` · **测试**：`Tests/CLITests/ArgParserTests.swift`
