# v2.2 — 服务集成

5 项 REQ（P0: 2，P1: 2，P2: 1）

> 卡片格式（OSS · BDD）：用户故事 + 验收标准（Given/When/Then）+ 实现/测试溯源。金标准见 v0.1-index-core.md。

> **v2.2 在 v2.1 基础上增加外部集成能力**。HTTP 本地搜索服务、URL Scheme、Apple Shortcuts、AppleScript 四种集成方式，让 DeepFinder 可被其他应用和自动化工具调用。所有集成通过 IPC 连接 daemon，复用现有搜索能力。

### REQ-2.2-01 HTTPSearchService（本地 HTTP 搜索服务）✅ 🖥️ P0

**用户故事**

作为「开发者 / 自动化工具」，我希望通过本地 HTTP 接口发起搜索并拿回 JSON 结果，以便用浏览器、curl、fetch 等任意客户端将 DeepFinder 集成进其他应用与脚本。

**验收标准（Given / When / Then）**

- **AC1 — 搜索接口**
  - **Given** daemon 已启动且 HTTP 服务开启
  - **When** 发起 `GET /search?q=keyword&limit=20&offset=0`
  - **Then** 返回标准 JSON `{ "query": "...", "results": [...], "total": N, "offset": N, "limit": N }`

- **AC2 — 统计与健康检查**
  - **Given** HTTP 服务开启
  - **When** 发起 `GET /stats` 或 `GET /health`
  - **Then** 分别返回索引统计与健康检查结果

- **AC3 — 实现约束（零外部依赖）**
  - **Given** 项目零外部依赖约束
  - **When** 实现 HTTP 服务
  - **Then** 使用 Network.framework（NWListener），不引入 SwiftNIO 或任何第三方库

- **AC4 — 端口可配置**
  - **Given** 默认端口 7654
  - **When** 通过 `deepfinder --serve --port 8080` 启动
  - **Then** HTTP 服务在新端口监听

- **AC5 — 服务开关**
  - **Given** HTTP 服务默认不启动
  - **When** 执行 `deepfinder --serve`（CLI 模式）
  - **Then** 启动 daemon（如未运行）并开启 HTTP 服务（HTTP 服务仅在 CLI `--serve` 模式下提供，不由 daemon 后台常驻）

- **AC6 — 仅本地监听**
  - **Given** HTTP 服务开启
  - **When** 绑定监听地址
  - **Then** 仅监听 localhost（127.0.0.1），不暴露到网络

- **AC7 — CORS**
  - **Given** 浏览器跨域请求（服务本身仅监听 localhost）
  - **When** 处理请求
  - **Then** 支持 CORS（响应携带 `Access-Control-Allow-Origin: *`、`Allow-Methods`、`Allow-Headers` 头，并处理 OPTIONS 预检返回 204）

**实现**：`Sources/Services/HTTPSearchService.swift` · **测试**：`Tests/ServicesTests/HTTPSearchServiceTests.swift`

### REQ-2.2-02 URL Scheme（deepfinder://）✅ 🖥️ P0

**用户故事**

作为「用户」，我希望在网页、邮件、笔记中点击 `deepfinder://search?q=report` 链接就能直接唤起 DeepFinder GUI 并执行搜索，以便从任何文档一键跳转到本地文件定位。

**验收标准（Given / When / Then）**

- **AC1 — Scheme 注册**
  - **Given** DeepFinder GUI 应用
  - **When** 查看 Info.plist
  - **Then** 已注册 `deepfinder://` URL Scheme

- **AC2 — 链接触发搜索**
  - **Given** GUI 已运行
  - **When** 用户点击 `deepfinder://search?q=keyword&limit=20` 链接
  - **Then** macOS 打开 DeepFinder GUI，自动打开搜索面板、填入查询并执行搜索

- **AC3 — 参数传递**
  - **Given** 一个合法的 deepfinder:// URL
  - **When** GUI 解析参数
  - **Then** 支持 q（查询）、limit（数量）、filter（过滤器）的传递

- **AC4 — GUI 未运行时启动**
  - **Given** GUI 未运行
  - **When** 用户点击 deepfinder:// 链接
  - **Then** macOS 启动 GUI 后再处理该 URL

- **AC5 — 无效 URL 容错**
  - **Given** 无效或不支持的 deepfinder:// URL
  - **When** GUI 接收
  - **Then** 忽略该 URL 且不崩溃

**实现**：`Sources/Services/URLSchemeHandler.swift` · **测试**：`Tests/ServicesTests/URLSchemeHandlerTests.swift`

### REQ-2.2-03 AppIntents（Apple Shortcuts）✅ 🖥️ P1

**用户故事**

作为「自动化用户」，我希望在 Shortcuts app 中调用 DeepFinder 搜索（例如"每天早上 9 点搜索 Downloads 中超过 7 天的大文件"），以便把本地文件搜索纳入跨应用的自动化流水线，并让结果在后续步骤中被复用。

**验收标准（Given / When / Then）**

- **AC1 — SearchFilesIntent**
  - **Given** 一个已实现的 `SearchFilesIntent` AppIntent
  - **When** 在 Shortcuts 中输入 query 字符串调用
  - **Then** 返回匹配的文件路径数组

- **AC2 — GetFileInfoIntent**
  - **Given** 一个已实现的 `GetFileInfoIntent` AppIntent
  - **When** 输入文件路径调用
  - **Then** 返回该文件的元数据

- **AC3 — Shortcuts 可发现**
  - **Given** Shortcuts app
  - **When** 用户创建自动化并搜索 DeepFinder 动作
  - **Then** 可搜索文件并获取结果，结果可在后续步骤中使用（打开、复制路径等）

- **AC4 — Intent 参数**
  - **Given** AppIntent 参数定义
  - **When** 查看参数
  - **Then** SearchFilesIntent 含 query（必填）、limit（可选，默认 20）；GetFileInfoIntent 含 path（必填）

**实现**：`Sources/Services/SearchIntent.swift` · **测试**：`Tests/ServicesTests/SearchIntentTests.swift`

### REQ-2.2-04 AppleScript 支持 ✅ 🖥️ P1

**用户故事**

作为「Mac 自动化用户」，我希望用 AppleScript 调用 DeepFinder 搜索（`tell application "DeepFinder" to search "report"`）并拿回路径列表，以便与其他 Mac 自动化工具（如 Alfred、Keyboard Maestro、Hammerspoon）集成。

**验收标准（Given / When / Then）**

- **AC1 — search 命令**
  - **Given** 一个 `NSScriptCommand` 子类处理 `search` 命令
  - **When** AppleScript 执行 `search "query"`（直接参数为查询字符串）
  - **Then** 返回文件路径列表

- **AC2 — 脚本字典**
  - **Given** sdef（Scripting Definition）文件
  - **When** AppleScript 引用 DeepFinder
  - **Then** 已定义脚本字典，可被脚本编辑器识别

- **AC3 — daemon 未运行容错**
  - **Given** daemon 未运行
  - **When** AppleScript 调用 search 命令
  - **Then** 返回空列表并输出警告（不报错中断）

**实现**：`Sources/Services/SearchScriptCommand.swift` · **测试**：`Tests/ServicesTests/SearchScriptCommandTests.swift`

### REQ-2.2-05 CLI --serve 模式 ✅ 🖥️ P2

**用户故事**

作为「服务器 / 无 GUI 环境用户」，我希望用 `deepfinder --serve` 启动 daemon + HTTP 搜索服务而不依赖 GUI，以便在 headless 机器上也能通过 HTTP 搜索文件。

**验收标准（Given / When / Then）**

- **AC1 — 启动 daemon + HTTP**
  - **Given** daemon 未运行
  - **When** 执行 `deepfinder --serve`
  - **Then** 启动 daemon（如未运行）并开启 HTTP 服务（无 GUI）

- **AC2 — 自定义端口**
  - **Given** `--serve` 模式
  - **When** 附加 `--port` 参数
  - **Then** 覆盖默认端口在该端口监听

- **AC3 — 启动提示**
  - **Given** `--serve` 成功启动
  - **When** 服务就绪
  - **Then** 输出 `HTTP search service running on http://localhost:{port}` 启动信息

- **AC4 — 优雅关闭**
  - **Given** `--serve` 正在运行
  - **When** 用户按下 Ctrl+C
  - **Then** 优雅关闭（flush 索引 + 停止 HTTP 服务）

**实现**：`Sources/CLI/ServeMode.swift` · **测试**：`Tests/ServicesTests/ServeModeTests.swift`
