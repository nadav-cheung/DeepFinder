# v2.2 — 服务集成

5 项 REQ (P0: 2, P1: 2, P2: 1)

> **v2.2 在 v2.1 基础上增加外部集成能力**。HTTP 本地搜索服务、URL Scheme、Apple Shortcuts、AppleScript 四种集成方式，让 DeepFinder 可被其他应用和自动化工具调用。所有集成通过 IPC 连接 daemon，复用现有搜索能力。

---

### REQ-2.2-01 HTTPSearchService（本地 HTTP 搜索服务）📋 🖥️ P0

**用户场景**：用户在浏览器中打开 `http://localhost:7654/?q=report` → 浏览器显示搜索结果 JSON。开发者可用 curl/fetch 集成。

**操作流程**：
1. Daemon 启动时自动开启 HTTP 服务（默认端口 7654，可配置）
2. GET `/search?q=keyword&limit=20&offset=0` → 返回 JSON 结果
3. GET `/stats` → 返回索引统计
4. GET `/health` → 健康检查

**验收标准**：
- [ ] SwiftNIO 或 NWListener 实现 HTTP 服务（零外部依赖，纯 Network.framework）
- [ ] 端口可配置（`config set http.port 8080`），默认 7654
- [ ] HTTP 服务可通过 config 开关（`config set http.enabled true`，默认关闭）
- [ ] 仅监听 localhost（127.0.0.1），不暴露到网络
- [ ] CORS 支持（允许 localhost 来源）
- [ ] 请求超时 30s
- [ ] 返回标准 JSON：`{ "query": "...", "results": [...], "total": N, "offset": N, "limit": N }`

### REQ-2.2-02 URL Scheme（deepfinder://）📋 🖥️ P0

**用户场景**：用户点击 `deepfinder://search?q=report` 链接 → DeepFinder GUI 自动弹出搜索面板并执行搜索。网页/邮件/笔记中可直接链接到搜索。

**操作流程**：
1. 系统注册 `deepfinder://` URL Scheme
2. 用户点击 deepfinder://search?q=keyword 链接
3. macOS 打开 DeepFinder GUI
4. GUI 接收 URL → 解析参数 → 填入搜索框 → 执行搜索

**验收标准**：
- [ ] 注册 `deepfinder://` URL Scheme（Info.plist）
- [ ] 支持 URL：`deepfinder://search?q=keyword&limit=20`
- [ ] GUI 接收 URL 后自动打开搜索面板并执行搜索
- [ ] 参数传递：q（查询）、limit（数量）、filter（过滤器）
- [ ] 无效 URL 忽略（不崩溃）
- [ ] GUI 未运行时启动 GUI 再处理

### REQ-2.2-03 AppIntents（Apple Shortcuts）📋 🖥️ P1

**用户场景**：用户在 Shortcuts app 中创建自动化："每天早上 9 点搜索 Downloads 中超过 7 天的大文件" → Shortcuts 调用 DeepFinder 搜索 → 获取结果列表。

**验收标准**：
- [ ] `AppIntent` 实现 `SearchFilesIntent`：输入 query 字符串，返回文件路径数组
- [ ] `AppIntent` 实现 `GetFileInfoIntent`：输入文件路径，返回文件元数据
- [ ] Shortcuts 中可搜索文件并获取结果
- [ ] 结果可在 Shortcuts 后续步骤中使用（打开、复制路径等）
- [ ] Intent 参数：query（必填）、limit（可选，默认 20）、filter（可选）

---

### REQ-2.2-04 AppleScript 支持 ✅ 🖥️ P1

**用户场景**：用户写 AppleScript 自动化：`tell application "DeepFinder" to search "report"` → 获取搜索结果。可与其他 Mac 自动化工具集成。

**验收标准**：
- [ ] 实现 `NSScriptCommand` 子类处理 `search` 命令
- [ ] AppleScript 接口：`search "query" with limit N`
- [ ] 返回文件路径列表
- [ ] sdef 定义脚本字典
- [ ] 错误处理：daemon 未运行时返回空列表并输出警告

---

### REQ-2.2-05 CLI --serve 模式 ✅ 🖥️ P2

**用户场景**：无 GUI 的服务器环境也能通过 HTTP 搜索。`deepfinder --serve` 启动 daemon + HTTP 服务（无 GUI）。

**验收标准**：
- [ ] CLI `deepfinder --serve` 启动 daemon（如未运行）+ HTTP 服务
- [ ] `--port` 参数覆盖默认端口
- [ ] Ctrl+C 优雅关闭（flush 索引 + 停止 HTTP）
- [ ] 输出 "HTTP search service running on http://localhost:{port}" 启动信息
