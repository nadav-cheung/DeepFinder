### REQ-3.0-01 AIModelProvider 协议 ✅ 🖥️ P0

**用户故事**

作为「开发者」，我希望有一个统一的 AI 模型提供者协议，以便未来接入新模型（如 GPT-5、Claude）只需新增一个 Provider 文件，不修改现有代码。

**验收标准（Given / When / Then）**

- **AC1 — 协议形态**
  - **Given** `AIModelProvider: Sendable` 协议
  - **When** 审视其接口
  - **Then** 包含 `name`、`capabilities`、`complete()`、`translateToSearchSyntax()`

- **AC2 — 能力枚举**
  - **Given** `AICapability` 枚举
  - **When** 列举其成员
  - **Then** 覆盖 `textToSearch` / `resultSummary` / `querySuggestion` / `intentAnalysis` / `localVision` / `localSpeech`

- **AC3 — 隐私边界**
  - **Given** `AIContext` 类型
  - **When** 组装发送到云端的内容
  - **Then** 只包含元数据（`FileMetadataSummary`），不含任何文件内容

- **AC4 — 流式响应**
  - **Given** 调用 `complete()`
  - **When** 模型返回响应
  - **Then** 返回 `AsyncThrowingStream<String, Error>` 流式输出

- **AC5 — 并发安全**
  - **Given** Swift 6 严格并发模式
  - **When** 跨 actor / 跨任务使用协议实现
  - **Then** 编译通过，无并发告警

**实现**：`Sources/AI/AIModelProvider.swift` · **测试**：`Tests/AITests/AIModelProviderTests.swift`

### REQ-3.0-02 Privacy Boundary（隐私边界）✅ 🖥️ P0

**用户故事**

作为「用户」，我希望使用 AI 功能时文件内容、图片像素、任何二进制数据永不离开本机，以便完全放心地使用 AI 搜索；并可通过 CLI `:config ai.data_preview true` 或 Settings 预览实际外传数据。

**验收标准（Given / When / Then）**

- **AC1 — 元数据字段限定**
  - **Given** `FileMetadataSummary`
  - **When** 序列化发送到云端
  - **Then** 只含 `name`、`path`（脱敏）、`size`、`modifiedAt`、`extension`、`localTags`

- **AC2 — 路径脱敏**
  - **Given** 路径脱敏默认开启
  - **When** 序列化 `/Users/nadav/...`
  - **Then** 输出为 `~/...`

- **AC3 — 无内容字段**
  - **Given** `FileMetadataSummary`
  - **When** 检查其字段
  - **Then** 无文件内容字段、无缩略图字段、无二进制数据字段

- **AC4 — 编译期强制**
  - **Given** 类型系统
  - **When** `AIContext` 引用文件信息
  - **Then** 只能引用 `FileMetadataSummary`，不能引用 `FileRecord` 的其他字段

- **AC5 — 数据预览**
  - **Given** 用户想查看实际发送的数据
  - **When** 运行 `deepfinder config get ai.data_preview`（v2.0 GUI: Settings > AI）
  - **Then** 展示实际发送的 JSON 样例

**实现**：`Sources/AI/FileMetadataSummary.swift`、`Sources/AI/AIContext.swift` · **测试**：`Tests/AITests/FileMetadataSummaryTests.swift`
