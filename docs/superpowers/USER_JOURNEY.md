## 1. 概述

DeepFinder 是一款 macOS 文件搜索工具，自建索引、零外部依赖、免费开源，替代 Spotlight + Alfred + HoudahSpot + EasyFind。本文档是内部产品规划工具，从用户视角绘制从发现到推荐的完整旅程，识别摩擦点、流失风险和机会空间。

### 1.1 用户画像速览

| 画像 | 描述 | 核心诉求 | 日搜索频率 | 典型场景 |
|------|------|---------|-----------|---------|
| **开发者** | 写代码、管项目、找配置文件和日志 | 速度、精确、可脚本化 | 50-200 次 | `deepfinder "AppDelegate.swift"` / `--json` 管道 |
| **知识工作者** | 管文档、PDF、设计稿、媒体文件 | 可靠、直观、Quick Look | 10-30 次 | GUI 唤起 → 输入文件名 → Space 预览 → Enter 打开 |
| **高级 Mac 用户** | 管大量文件、追求效率工具 | 快捷键、自定义、无阻碍 | 30-80 次 | ⌃⌘K → 搜索 → 自定义热键排除目录 |

### 1.2 方法论说明

本旅程地图遵循 NNGroup 用户旅程方法论（旅程 = 高层次、跨渠道、长期关系；流程 = 产品内具体交互），但**刻意偏离了 NNGroup "one map per persona" 规则**——本图在一份文档中覆盖 3 个画像。理由是：(1) DeepFinder 三个画像的使用模式高度重叠（核心循环相同，摩擦差异仅在配置和高级功能阶段）；(2) 小团队维护三份独立地图的边际收益不足以覆盖同步成本。摩擦矩阵（§3）按画像区分，确保画像间差异不被掩盖。

## 2. 旅程总览图

### 2.1 8 阶段漏斗

```
发现 ──→ 安装 ──→ 首次运行 ──→ 日常使用 ──→ 探索配置 ──→ 高级功能 ──→ 更新维护 ──→ 推荐传播
  ↓         ↓         ↓            ↓            ↓            ↓            ↓            ↓
理解价值   完成安装    Aha Moment   肌肉记忆      个性化       能力发现      持续运行      口碑传播
```

每个阶段的 UX 投入权重（总投入 100%）：

| 阶段 | 权重 | 阶段 | 权重 |
|------|------|------|------|
| 1. 发现 | 5% | 5. 探索配置 | 10% |
| 2. 安装 | 15% | 6. 高级功能 | 8% |
| 3. 首次运行 | 25% | 7. 更新维护 | 4% |
| 4. 日常使用 | 30% | 8. 推荐传播 | 3% |

### 2.2 摩擦矩阵

摩擦等级定义：**极低**（肌肉记忆，无意识完成）/ **低**（偶尔停顿，不影响流畅）/ **中**（需要查阅文档或多次尝试）/ **高**（可能导致放弃）。本矩阵替换了传统的 emoji 情绪弧线——开发者 200 次/天的体验是"流畅"而非"愉悦"，摩擦描述比情绪标签更准确。

| 阶段 | 开发者 | 知识工作者 | 高级 Mac 用户 |
|------|--------|-----------|-------------|
| **1. 发现** | **极低摩擦**：HN、GitHub Trending、Homebrew 搜索自然触达。价值主张 3 秒内理解 | **中摩擦**：需朋友推荐、博客评测或 YouTube 视频。信任门槛高——"又一个 Spotlight 替代品？" | **低摩擦**：效率工具社区（Reddit r/macapps、Mac Power Users）主动搜索并评估 |
| **2. 安装** | **极低摩擦**：`brew install` 一条命令。Homebrew 生态习惯 | **中摩擦**：DMG 拖拽 + macOS Gatekeeper 绕过。`brew` 不在其常用工作流中 | **低摩擦**：熟悉 Homebrew 或手动安装。可能遇到自签证书信任问题 |
| **3. 首次运行** | **低摩擦**：FDA 权限授予熟悉，但 CLI 首次索引等待可能打断节奏。期望<100ms 响应被索引构建时间打破 | **高摩擦**：FDA 权限引导陌生且吓人。"为什么这个免费工具需要看我的所有文件？"——信任危机时刻 | **低摩擦**：理解 FDA 必要性，但可能要求更细粒度的权限控制 |
| **4. 日常使用** | **极低摩擦**：肌肉记忆，50-200 次/天。痛点：复杂布尔语法错误、FDA 缺失导致静默遗漏文件 | **低摩擦**：10-30 次/天。痛点：搜索语法不熟悉、不知 Quick Look 可用、GUI/CLI 切换困惑 | **低摩擦**：30-80 次/天。痛点：默认热键与其他工具冲突、结果排序不符合习惯 |
| **5. 探索配置** | **低摩擦**：明确知道要排除 `node_modules`、`.git`、`DerivedData`。通过配置文件或 CLI 命令完成 | **中摩擦**：默认配置够用，不主动探索。可能错过对效率有显著提升的配置项 | **低摩擦**：精细定制热键、外观、结果数量。期望即时预览配置变更效果 |
| **6. 高级功能** | **低摩擦**：自然发现 NL 搜索、正则、脚本化。CLI 管道是日常操作 | **中摩擦**：不知道 NL 搜索可用、AI 隐私顾虑、语音输入使用场景不清 | **低摩擦**：主动探索 AI 功能边界，但可能对云服务有隐私顾虑 |
| **7. 更新维护** | **极低摩擦**：`brew upgrade` 例行操作。失败时直接 `brew reinstall` | **中摩擦**：macOS 升级后权限断裂（FDA 重置）是已知痛点。不主动关注版本更新 | **低摩擦**：关注 Release Notes，根据变更决定是否升级 |
| **8. 推荐传播** | **低摩擦**：HN 评论、Twitter、团队 Slack 自然推荐。可附上 benchmark 数据增强说服力 | **中摩擦**：推荐门槛高——"除非比 Spotlight 明显好，否则不推荐"。需要明显的差异化体验 | **低摩擦**：效率工具博客、Reddit 评测、YouTube 演示。喜欢用对比数据推荐 |

## 3. 阶段详解

### 3.1 发现

**用户目标**：理解 DeepFinder 是什么，判断是否值得尝试。

**关键行为**：在 GitHub Trending、Hacker News、Reddit r/macapps、Homebrew 搜索或效率工具评测博客中首次接触。用户在 3 秒内决定是否继续了解。

**关键接触点**：
- GitHub README（`nadav/deepfinder`）——价值主张必须在首屏折叠线之上
- Homebrew 描述行——`brew info deepfinder` 输出的一句话摘要
- 社区帖子标题——"Show HN: DeepFinder — Find any file on your Mac, instantly"

**成功指标**：3 秒内理解价值主张。测量方法：用户测试。

**测量方法**：用户测试（5 人，展示 README 前 3 段，计时理解价值主张）。

**常见流失点**：价值主张模糊——"又一个文件搜索工具？我有 Spotlight。"README 首段未陈述架构独立性（自建索引 vs 依赖 Spotlight），用户未感知差异化。

**无障碍说明**：README 中的终端截图必须配 alt text，方便屏幕阅读器用户理解 CLI 输出。

---

### 3.2 安装

**用户目标**：60 秒内完成安装，准备首次搜索。

**关键行为**：
- Homebrew 路径：`brew install nadav/deepfinder/deepfinder`（约 10 秒下载 + 编译缓存命中时）
- DMG 路径：下载 `.dmg` → 双击挂载 → 拖拽到 Applications → 首次启动时绕过 Gatekeeper（右键→打开）

**关键接触点**：
- Homebrew Formula 可用性——`brew search deepfinder` 能找到
- DMG 下载页——GitHub Releases 页面
- macOS Gatekeeper 对话框——"DeepFinder 是从互联网下载的应用程序。确定要打开吗？"

**成功指标**：安装到首次搜索 ≤ 60 秒（Homebrew 路径，不含索引构建）。测量方法：手动计时。

**测量方法**：手动计时（5 次 Homebrew 安装，取中位数）。

**常见流失点**：
- Homebrew 404（Formula 未收录到 core tap）——用户对 `nadav/deepfinder/deepfinder` 路径困惑
- macOS Gatekeeper 阻止——未公证的 DMG 无法双击打开，需右键→打开
- `brew install` 后找不到 `deepfinder` 命令——Homebrew 未在 PATH 中

**无障碍说明**：安装说明中的终端命令必须可直接复制粘贴——不使用零宽字符、不包含引号混淆。

---

### 3.3 首次运行

**用户目标**：完成首次搜索，体验 Aha Moment（< 100ms 返回结果）。

**关键行为**：
1. 首次启动触发 FDA 权限引导（deferred 到首次搜索时）
2. 索引构建（首次扫描全盘，约 10-30 秒，取决于文件数量）
3. 输入第一个搜索词
4. 看到搜索结果几乎即时出现（< 100ms 查询延迟）

**关键接触点**：
- FDA 权限对话框——macOS 系统弹窗："DeepFinder 需要访问其他应用程序的数据"。用户在此刻做出信任决策
- 索引状态指示器——终端显示 "Indexing… 45% (123,456 files)" 或 GUI 进度条
- 首个搜索结果——用户输入查询后看到的第一个结果列表

**成功指标**：Aha Moment < 100ms（索引就绪后的查询延迟）。测量方法：Instruments 时间剖析。

**测量方法**：Instruments 时间剖析（测量从 `Enter` 按下到首个结果渲染的时间间隔）。

**常见流失点**：
- FDA 权限请求文案不足——用户看到"需要访问所有文件"感到恐惧，拒绝授权。缺少解释文案（"DeepFinder 需要完全磁盘访问才能搜索你的文件。所有索引数据完全保留在本机，不上传云端"）
- 首次索引时间过长——用户期望即时可用，10-30 秒等待可能触发"卡住了"的误判
- CLI 用户第一次输入 `deepfinder` 无参数时进入 REPL 模式——与预期"直接搜索"行为冲突

**无障碍说明**：
- **VoiceOver**：索引进度指示器必须提供无障碍标签（"正在索引您的文件，已完成 45%，预计还需 15 秒"）。FDA 权限引导文案必须可被 VoiceOver 朗读
- **键盘导航**：权限引导对话框必须支持 Tab/Enter 完整操作

### 3.4 日常使用（Search & Act）

**用户目标**：找到目标文件并执行操作（打开/预览/显示位置）。

**关键行为**：
1. ⌃⌘K 唤起搜索面板（GUI）或执行 `deepfinder "query"`（CLI）
2. 输入搜索字符，实时过滤结果
3. ↑↓ 浏览结果，Space 预览，Enter 打开
4. Esc 关闭面板，焦点回原应用

**关键接触点**（按频率排序）：
1. ⌃⌘K 唤起 —— 每天 10-200 次，必须 < 100ms
2. 搜索输入 —— 每字符 < 50ms 更新结果
3. 结果行高亮 + 预览 —— 即时反馈
4. Enter 打开文件 —— 调用默认应用
5. Esc 关闭 —— < 100ms 消失

**成功指标**：唤起 → 操作完成 < 2 秒。测量方法：Instruments 时间剖析。

**测量方法**：Instruments 时间剖析（测量从热键按下到文件打开完成的时间间隔）。

**搜索失败子事件**（评审 H-5：这是 DeepFinder 的 #1 竞争优势时刻）：

当用户搜索无结果时，经历以下微旅程：

```
用户动作：输入已知存在的文件名 → 无结果
   ↓
系统响应：显示友好提示："未找到 'report-2026.xlsx'。可能原因：文件在排除目录中 / 完全磁盘访问未启用 / 索引正在重建"
   ↓
用户情绪：焦虑（"文件丢了？"）→ 系统给出可操作建议 → 尝试修复（检查 FDA / 排除列表）
   ↓
验证：文件找到 → 信任加深（"这个工具比 Spotlight 可靠——它告诉我为什么找不到"）
```

**Spotlight 对比**：Spotlight 在此场景下静默返回空结果——用户不知道是文件不存在、未索引、还是权限问题。DeepFinder 的差异化优势在于：**告诉用户为什么找不到，并给出修复路径**。这是将负面体验转化为信任增强时刻的关键设计。

**测量方法（搜索失败）**：用户测试（故意排除目录后搜索，观察用户是否理解原因并自行修复）。

**常见流失点**：
- FDA 权限在 macOS 更新后被重置——用户不知道为何突然搜不到文件
- 默认热键与其他应用冲突（⌃⌘K 被 VS Code、Obsidian 占用）
- 外置卷卸载后搜索结果中残留死链接

**无障碍说明**：
- **全键盘操作**：搜索面板所有功能（唤起、输入、浏览、预览、打开、关闭）必须仅通过键盘完成。Tab 在搜索框和结果列表之间切换，Enter 打开、Space 预览、Esc 关闭。这是 UX-M04 的要求
- **Reduce Motion**：当系统「减弱动态效果」开启时，面板出现/消失动画替换为即时淡入淡出（禁用弹性动画和位移效果），符合 UX-M05

### 3.5 探索配置

**用户目标**：按个人习惯定制 DeepFinder，让工具适配自己的工作流。

**关键行为**：
- 排除不需要搜索的目录（`node_modules`、`.git`、`DerivedData`、`Library/Caches`）
- 自定义全局热键（从 ⌃⌘K 改为其他组合）
- 调整结果数量（默认 100 条/页）
- 开关 AI 功能（隐私/性能权衡）
- 搜索行为偏好（默认大小写不敏感、路径显示方式）

**关键接触点**：
- CLI：`deepfinder config` 交互式配置或 `~/.deep-finder/config.toml` 直接编辑
- GUI：设置面板（Preferences Window），实时预览配置变更效果

**成功指标**：配置变更即时生效（无需重启 daemon）。测量方法：功能测试。

**测量方法**：功能测试（修改每个配置项后执行对应搜索，验证即时生效）。

**常见流失点**：
- 配置键命名不直观——用户不知道哪个键对应哪个行为
- 缺少配置变更预览——改了排除路径不知道是否正确匹配
- 配置文件格式错误静默回退——用户的错误配置被忽略，没有错误提示
- TOML 语法错误导致 daemon 启动失败——缺少配置验证和友好错误信息

**无障碍说明**：设置面板所有控件必须支持标准 macOS 无障碍 API（NSAccessibility），可被 VoiceOver 朗读标签和值。

### 3.6 高级功能

**用户目标**：发现并使用 Natural Language 搜索、内容搜索、语音输入、AI 语义分组等进阶能力。

**关键行为**：
- NL 搜索："找到上个月修改的 PDF 发票"
- 正则搜索：`/Invoice_\d{4}\.pdf/`
- 内容搜索：搜索文件内容中的关键词（PDF、Office 文档、纯文本）
- 语音输入：点击麦克风按钮口述搜索条件
- AI 语义搜索：按概念而非文件名搜索
- 脚本化：`deepfinder --json "query" | jq`

**关键接触点**：
- 搜索框 placeholder 提示（"试试：'上个月的 PDF 发票'"）
- AI 功能开关和 API Key 配置
- 语音输入按钮（GUI）和 `:voice` 命令（CLI）
- `--json` 和 `--0` 输出模式的文档

**成功指标**：功能可发现性（用户在不阅读文档的情况下发现至少 1 项高级功能）。测量方法：用户测试。

**测量方法**：用户测试（5 人，观察首次使用后 5 分钟内是否尝试 NL 搜索或语音输入）。

**常见流失点**：
- AI 功能默认关闭——用户不知道这些功能存在（可发现性问题）
- 隐私顾虑——用户不确定 AI 搜索是否会上传文件内容到云端
- 语音输入在嘈杂环境中误触发
- NL 搜索翻译错误——复杂查询被错误解析，结果完全不相关

**无障碍说明**：
- **VoiceOver**：语音输入按钮必须提供无障碍标签（"语音搜索：点击后说话"），录制状态必须有听觉反馈（"正在听取…"）
- **Reduce Motion**：AI Intelligence Glow 动画在 Reduce Motion 开启时降级为静态渐变

### 3.7 更新维护

**用户目标**：保持 DeepFinder 最新版本，持续正常运行。

**关键行为**：
- `brew upgrade deepfinder`（Homebrew 用户，例行操作）
- Sparkle 自动检查更新（DMG 用户）
- macOS 系统升级后验证权限是否正常
- 索引重建触发（版本升级后可能需要）

**关键接触点**：
- Homebrew 更新输出
- Sparkle 更新弹窗
- 版本升级后的首次启动（可能触发索引重建）

**成功指标**：静默更新，不打断用户工作流。测量方法：发布监控（检查升级成功率）。

**测量方法**：发布监控（Homebrew analytics 安装量趋势 + GitHub Releases 下载量，监控异常下降）。

**重新激活子事件**（评审 M-2：macOS 升级后权限断裂是反复出现的已知场景）：

macOS 大版本升级（如 macOS 26→27）后，系统可能静默重置 FDA 权限列表。用户经历：

```
1. 升级 macOS，日常使用一切正常
2. 某天搜索文件 → 结果显著减少（关键文件缺失）
3. 用户可能数天不察觉，直到找不到特定文件
4. 打开 DeepFinder → 看到 "⚠️ 索引不完整：完全磁盘访问权限未启用"
5. 点击提示 → 跳转系统设置 → 重新授权
6. 索引重建（约 10-30 秒）→ 恢复正常
```

**关键设计**：在搜索结果区域顶部用醒目但不打扰的方式显示索引健康状态（UX-M06）。不要弹出模态对话框——用户来搜索，不是来处理错误。

**常见流失点**：
- macOS 升级后权限静默丢失 → 用户认为 DeepFinder 坏了
- `brew upgrade` 后 daemon 未自动重启 → 新版本未生效
- 版本升级后索引格式不兼容 → 需要完全重建，用户困惑

**无障碍说明**：索引健康警告必须可被 VoiceOver 朗读，并提供键盘可访问的"前往设置"操作按钮。

### 3.8 推荐传播

**用户目标**：主动将 DeepFinder 推荐给同事、朋友、社区。

**关键行为**：

## 6. 竞品对标摘要

DeepFinder 的架构独立性（自建索引、不依赖 Spotlight）是核心竞争壁垒。与竞品的详细对比见 [竞品对比文档](../COMPARISON.md)，UX 层面的竞品需求分析见 [UX Requirements - 竞品矩阵](specs/ux/2026-06-03-user-experience-requirements.md#21-竞品矩阵)。

**关键差异化**：Spotlight/Alfred/Raycast 均依赖 macOS Spotlight 索引（不可靠，静默遗漏文件），EasyFind/Find Any File 使用文件系统暴力搜索（秒级响应），HoudahSpot 功能强大但无免费版本。DeepFinder 是唯一同时提供自建索引（毫秒级响应）+ 免费开源 + CLI REPL + AI 语义搜索 + 全模态媒体元数据的工具。

竞品数据由 `researcher` 每季度更新一次，重大竞品版本发布时即时更新。来源：竞品官网、社区论坛、App Store 版本记录、Homebrew analytics。
